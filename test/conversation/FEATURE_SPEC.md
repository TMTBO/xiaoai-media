# 对话记录功能规格说明

## 功能概述

对话记录功能允许用户通过 Web 界面查看与小爱音箱的历史对话记录。

## 功能需求

### 1. 后端 API

#### 1.1 获取对话记录
- **端点**: `GET /api/command/conversation`
- **参数**: 
  - `device_id` (可选): 设备 ID
- **响应**:
```json
{
  "conversations": [
    {
      "timestamp_ms": 1773740151066,
      "question": "用户问题",
      "content": "小爱回答"
    }
  ]
}
```

#### 1.2 数据来源
- 使用小米对话 API: `https://userprofile.mina.mi.com/device_profile/v2/conversation`
- 参数:
  - `source`: dialogu
  - `hardware`: 设备型号
  - `timestamp`: 当前时间戳
  - `limit`: 记录数量（默认 10）

#### 1.3 认证方式
- Cookie 认证
- 使用 deviceId 作为 Cookie

### 2. 前端界面

#### 2.1 页面布局
- 顶部操作栏
  - 设备选择器
  - 刷新按钮
  - 查询按钮
- 对话时间轴
  - 时间戳
  - 用户问题（蓝色图标）
  - 小爱回答（绿色图标）

#### 2.2 交互功能
- 设备选择
- 设备列表刷新
- 对话记录查询
- 加载状态显示
- 错误提示
- 空状态提示

#### 2.3 时间格式化
- 今天: "今天 HH:MM:SS"
- 昨天: "昨天 HH:MM:SS"
- 本周: "周X HH:MM:SS"
- 其他: "YYYY-MM-DD HH:MM:SS"

## 技术实现

### 1. 后端技术栈
- Python 3.11+
- FastAPI
- aiohttp
- miservice

### 2. 前端技术栈
- Vue 3
- TypeScript
- Element Plus
- Vite

### 3. 数据流

```
用户点击查询
    ↓
前端发送 API 请求
    ↓
后端接收请求
    ↓
调用小米对话 API
    ↓
解析响应数据
    ↓
格式化为标准格式
    ↓
返回给前端
    ↓
前端展示对话记录
```

## 错误处理

### 1. 后端错误
- HTTP 错误: 返回 502 状态码
- JSON 解析错误: 返回空列表
- 网络错误: 记录日志并抛出异常

### 2. 前端错误
- API 错误: 显示错误提示
- 网络错误: 显示错误提示
- 空数据: 显示空状态提示

## 性能要求

- API 响应时间: < 500ms
- 前端渲染时间: < 100ms
- 支持设备数: 无限制
- 对话记录数: 10 条（可配置）

## 安全性

- 使用 HTTPS 通信
- Cookie 认证
- 不存储敏感信息
- 仅读取权限

## 兼容性

### 浏览器
- Chrome/Edge 90+
- Firefox 88+
- Safari 14+

### 设备
- 支持所有小爱音箱型号
- 自动识别设备硬件型号

## 限制

1. 对话记录由小米服务器提供，可能存在延迟
2. 不同设备型号返回的记录数量可能不同
3. 需要有效的小米账号认证
4. 不支持实时更新，需要手动刷新
