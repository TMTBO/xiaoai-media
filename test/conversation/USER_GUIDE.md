## 快速开始

### 访问页面

在浏览器中打开应用后，点击左侧导航栏的"对话记录"菜单项，或直接访问：
```
http://localhost:5173/conversation
```

### 基本操作

1. 选择设备（可选）
2. 点击"查询对话"按钮
3. 查看对话记录

## 功能说明

### 1. 设备选择

支持查看不同小爱音箱的对话记录：

- 点击设备选择器查看所有设备
- 选择要查询的设备
- 如果不选择，将使用默认设备

### 2. 对话展示

对话记录以时间轴形式展示，包含：

**时间戳**
- 今天的对话：显示"今天 HH:MM:SS"
- 昨天的对话：显示"昨天 HH:MM:SS"
- 一周内：显示"周X HH:MM:SS"
- 更早：显示完整日期时间

**用户问题**（蓝色图标）
- 显示你对小爱说的话

**小爱回答**（绿色图标）
- 显示小爱的回复内容

### 3. 刷新功能

- 点击设备选择器旁的刷新按钮
- 重新加载设备列表
- 获取最新的设备信息

## 使用步骤

### 步骤 1: 配置账号

首次使用前，需要配置小米账号：

1. 点击左侧"配置管理"菜单
2. 填写小米账号和密码
3. 保存配置

### 步骤 2: 查看对话

1. 点击左侧"对话记录"菜单
2. （可选）选择要查询的设备
3. 点击"查询对话"按钮
4. 等待加载完成
5. 查看对话记录

### 步骤 3: 切换设备

如果有多个小爱音箱：

1. 点击设备选择器
2. 从列表中选择其他设备
3. 点击"查询对话"按钮
4. 查看该设备的对话记录

## 常见问题

### Q1: 为什么看不到对话记录？

**可能原因：**
1. 还没有配置小米账号
2. 设备最近没有对话
3. 网络连接问题
4. 账号认证失败

**解决方法：**
1. 前往"配置管理"页面配置小米账号
2. 对小爱说几句话，然后重新查询
3. 检查网络连接
4. 重新登录小米账号

### Q2: 为什么有些对话没有回答？

**原因：**
某些指令（如设置闹钟、控制智能家居）小爱可能只执行操作，不进行语音回复。

**示例：**
- "定三十分钟的闹钟" - 通常没有语音回复
- "打开客厅灯" - 可能只回复"好的"或不回复
- "今天天气怎么样" - 会有详细的语音回复

### Q3: 对话记录会实时更新吗？

**说明：**
对话记录不会自动刷新，需要手动点击"查询对话"按钮获取最新记录。

**建议：**
如果需要查看最新对话，请在与小爱对话后等待几秒钟，然后点击查询按钮。

### Q4: 可以查看多久之前的对话？

**说明：**
- 目前显示最近的 10 条对话记录
- 对话记录保留时长由小米服务器决定
- 通常可以查看最近几天的对话

### Q5: 为什么时间显示不准确？

**说明：**
时间基于你的本地时区显示，如果时区设置不正确，可能导致时间显示偏差。

**检查方法：**
1. 检查系统时区设置
2. 确认浏览器时区设置
3. 刷新页面重新加载

### Q6: 设备列表为空怎么办？

**可能原因：**
1. 小米账号未配置
2. 账号下没有绑定设备
3. 网络连接问题

**解决方法：**
1. 配置小米账号
2. 在米家 App 中绑定设备
3. 点击刷新按钮重新加载

## 使用技巧

### 技巧 1: 快速查看最新对话

1. 打开对话记录页面
2. 直接点击"查询对话"（使用默认设备）
3. 查看时间轴顶部的最新记录

### 技巧 2: 多设备管理

如果你有多个小爱音箱：

1. 为每个设备起一个容易识别的名字
2. 使用设备选择器快速切换
3. 分别查询各设备的对话记录
4. 了解每个设备的使用情况

### 技巧 3: 问题排查

如果小爱没有正确响应你的指令：

1. 查看对话记录
2. 确认小爱是否正确识别了你的问题
3. 如果识别错误，调整说话方式
4. 如果识别正确但没有回应，可能是功能限制

### 技巧 4: 了解使用习惯

通过对话记录可以：

1. 查看最常用的指令
2. 了解设备使用频率
3. 发现可能的问题
4. 优化使用方式

## 数据说明

### 对话记录来源

对话记录由小米服务器提供，通过以下 API 获取：
```
https://userprofile.mina.mi.com/device_profile/v2/conversation
```

### 数据更新

- 对话记录可能存在延迟（通常几秒到几分钟）
- 需要手动刷新获取最新数据
- 不同设备型号可能返回不同数量的记录

### 隐私说明

- 对话记录存储在小米服务器
- 本应用仅读取和展示，不存储对话内容
- 请妥善保管你的小米账号信息
- 不要在公共设备上使用

## 最佳实践

### 1. 定期查看

建议定期查看对话记录，了解：
- 设备使用情况
- 指令识别准确率
- 可能的问题

### 2. 保护隐私

- 不要在公共场所展示对话记录
- 定期更改小米账号密码
- 退出时清除浏览器缓存

### 3. 优化使用

根据对话记录：
- 调整说话方式
- 使用更准确的指令
- 避免识别错误的表达

## 技术支持

如遇到问题，请检查：

1. **后端服务**：确保后端 API 正常运行
   ```bash
   curl http://localhost:8000/api/command/conversation
   ```

2. **前端服务**：确保前端页面可以访问
   ```bash
   curl http://localhost:5173/conversation
   ```

3. **配置信息**：确保已正确配置小米账号

4. **网络连接**：确保可以访问小米服务器
