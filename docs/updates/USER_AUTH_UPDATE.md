# 用户认证功能更新说明

## 更新日期
2024-03-29

## 更新内容

本次更新为 XiaoAI Media 添加了完整的用户认证和权限管理系统。

## 新增功能

### 1. 用户登录系统
- 基于 JWT 的身份认证
- 安全的密码存储（SHA-256 哈希）
- 7天令牌有效期
- 自动登录状态管理

### 2. 用户管理
- 创建、编辑、删除用户
- 用户角色管理（管理员/普通用户）
- 密码修改功能
- 用户列表查看

### 3. 权限控制
- 基于角色的访问控制
- 管理员专属功能
- API 级别权限检查
- 前端路由守卫

### 4. 用户界面
- 现代化登录页面
- 用户管理页面
- 左侧栏用户信息显示
- 退出登录功能

## 默认账户

首次启动时会自动创建默认管理员账户：

- 用户名: `admin`
- 密码: `admin123`
- 角色: 管理员

**重要**: 首次登录后请立即修改默认密码！

## 安装步骤

### 1. 更新后端依赖

```bash
cd backend
pip install -e .
```

这会自动安装新增的 `pyjwt` 依赖。

### 2. 重启服务

```bash
# 如果使用 Docker
docker-compose restart

# 如果本地运行
cd backend
python run.py
```

### 3. 访问系统

打开浏览器访问系统，会自动跳转到登录页面。

使用默认账户登录：
- 用户名: `admin`
- 密码: `admin123`

## 升级注意事项

### 对现有用户的影响

1. **首次访问需要登录**
   - 所有用户首次访问时需要使用默认账户登录
   - 建议立即修改默认密码

2. **数据文件位置**
   - 用户数据存储在 `~/.xiaoai_media/users.json`
   - 首次启动时自动创建

3. **API 调用变化**
   - 所有 API 请求现在需要 Authorization 头
   - 前端已自动处理，无需手动修改

### Docker 用户

如果使用 Docker 部署，建议挂载数据目录以持久化用户数据：

```yaml
volumes:
  - ~/.xiaoai_media:/root/.xiaoai_media
```

## 功能使用

### 登录
1. 访问系统自动跳转到登录页
2. 输入用户名和密码
3. 点击"登录"按钮

### 修改密码
1. 以管理员身份登录
2. 点击左侧菜单"用户管理"
3. 找到要修改的用户，点击"编辑"
4. 输入新密码并保存

### 创建新用户
1. 以管理员身份登录
2. 点击左侧菜单"用户管理"
3. 点击"添加用户"按钮
4. 填写用户信息并保存

### 退出登录
点击左侧栏底部用户信息区域的菜单按钮，选择"退出登录"

## 安全建议

1. **立即修改默认密码**
2. **使用强密码**（包含字母、数字、特殊字符）
3. **定期更新密码**
4. **不要共享账户**
5. **生产环境修改 SECRET_KEY**

## 故障排除

### 无法登录
- 确认用户名和密码正确
- 检查 `~/.xiaoai_media/users.json` 文件是否存在
- 查看后端日志获取详细错误信息

### Token 过期
- Token 默认有效期为 7 天
- 过期后需要重新登录

### 忘记密码
- 删除 `~/.xiaoai_media/users.json` 文件
- 重启服务会重新创建默认管理员账户
