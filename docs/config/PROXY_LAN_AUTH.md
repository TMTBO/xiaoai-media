# 代理访问控制配置

## 功能说明

为了让小爱音箱等局域网设备能够无需身份认证直接访问音频代理接口，系统提供了局域网跳过认证的功能。

## 配置项

### PROXY_SKIP_AUTH_FOR_LAN

- 类型: `bool`
- 默认值: `True`
- 说明: 是否允许局域网内的设备跳过身份校验访问代理接口

### PROXY_LAN_NETWORKS

- 类型: `list[str]`
- 默认值: 
  ```python
  [
      "192.168.0.0/16",    # 常见家庭网络
      "10.0.0.0/8",        # 企业内网
      "172.16.0.0/12",     # 私有网络
      "127.0.0.0/8",       # 本地回环
  ]
  ```
- 说明: 局域网 IP 段配置（CIDR 格式），用于判断客户端 IP 是否在局域网范围内

## 使用方法

### 1. 在 user_config.py 中配置

```python
# 启用局域网跳过认证
PROXY_SKIP_AUTH_FOR_LAN = True

# 配置局域网 IP 段
PROXY_LAN_NETWORKS = [
    "192.168.0.0/16",    # 家庭网络
    "10.0.0.0/8",        # 企业内网
]
```

### 2. 在前端管理界面配置

1. 登录管理后台
2. 进入"配置管理"页面
3. 找到"代理访问控制"部分
4. 开启"局域网跳过认证"开关
5. 根据需要添加或删除局域网 IP 段
6. 点击"保存配置"

## 工作原理

1. 当请求到达 `/api/proxy/stream` 接口时，系统会检查客户端 IP 地址
2. 如果 `PROXY_SKIP_AUTH_FOR_LAN` 为 `True`，且客户端 IP 在 `PROXY_LAN_NETWORKS` 配置的范围内
3. 则跳过身份认证，允许直接访问
4. 否则，需要提供有效的 Bearer Token 进行身份认证

## 安全提示

⚠️ **重要**: 此功能仅应在受信任的局域网环境中启用

- 启用此功能后，局域网内的任何设备都可以访问代理接口
- 请确保您的局域网是安全的，不要在公网环境中使用
- 如果需要更高的安全性，建议关闭此功能，让所有请求都需要身份认证

## CIDR 格式说明

CIDR（无类别域间路由）是一种 IP 地址表示方法：

- `192.168.0.0/16`: 表示 192.168.0.0 到 192.168.255.255 的所有 IP
- `10.0.0.0/8`: 表示 10.0.0.0 到 10.255.255.255 的所有 IP
- `172.16.0.0/12`: 表示 172.16.0.0 到 172.31.255.255 的所有 IP
- `127.0.0.0/8`: 表示 127.0.0.0 到 127.255.255.255 的所有 IP（本地回环）

## 示例场景

### 场景 1: 家庭网络

```python
PROXY_SKIP_AUTH_FOR_LAN = True
PROXY_LAN_NETWORKS = ["192.168.1.0/24"]  # 只允许 192.168.1.x 网段
```

### 场景 2: 企业内网

```python
PROXY_SKIP_AUTH_FOR_LAN = True
PROXY_LAN_NETWORKS = [
    "10.0.0.0/8",        # 企业内网
    "172.16.0.0/12",     # VPN 网段
]
```

### 场景 3: 禁用局域网跳过认证

```python
PROXY_SKIP_AUTH_FOR_LAN = False
# 所有请求都需要身份认证，包括局域网内的请求
```

## 故障排查

### 问题: 小爱音箱无法播放音乐

1. 检查 `PROXY_SKIP_AUTH_FOR_LAN` 是否为 `True`
2. 检查小爱音箱的 IP 是否在 `PROXY_LAN_NETWORKS` 配置的范围内
3. 查看后端日志，确认请求是否被正确识别为局域网请求

### 问题: 局域网设备仍然需要认证

1. 确认配置已保存并重启服务
2. 检查客户端 IP 是否正确（可以在后端日志中查看）
3. 验证 CIDR 格式是否正确
