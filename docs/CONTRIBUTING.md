# 文档贡献指南

感谢你对 XiaoAI Media 文档的贡献！

---

## 📝 文档原则

### 1. 保持简洁
- 避免冗长的描述
- 使用清晰的标题和列表
- 提供代码示例而非长篇解释

### 2. 避免重复
- 一个主题只在一个地方详细说明
- 其他地方通过链接引用
- 定期检查和合并重复内容

### 3. 用户优先
- 从用户角度组织文档
- 提供快速开始指南
- 包含常见问题解答

### 4. 及时更新
- 代码变更时同步更新文档
- 标记过时的内容
- 定期审查和清理

## ✍️ 编写规范

### 文件命名
- 使用大写字母和下划线：`README.md`、`QUICK_START.md`
- 中文文档使用中文命名：`使用说明.md`
- 保持命名简洁明了

### 文档格式
- 使用 Markdown 格式
- 使用 `---` 分隔主要章节
- 使用 emoji 增强可读性（适度）

### 标题层级
```markdown
# 一级标题 - 文档标题
## 二级标题 - 主要章节
### 三级标题 - 子章节
#### 四级标题 - 细节说明
```

### 代码块
````markdown
```bash
# Shell 命令
make dev
```

```python
# Python 代码
from xiaoai_media import Client
```
````

### 链接
- 相对链接：`[配置指南](config/README.md)`
- 绝对链接：`[GitHub](https://github.com/...)`
- 锚点链接：`[快速开始](#快速开始)`

---

## 📋 文档类型

### 1. 快速开始（QUICK_START）
- 5-10 分钟完成
- 最少的配置步骤
- 立即可用的示例

### 2. 完整指南（README）
- 功能完整介绍
- 详细的配置说明
- 常见问题解答

### 3. API 参考（API_REFERENCE）
- 接口定义
- 参数说明
- 请求/响应示例

### 4. 故障排查（TROUBLESHOOTING）
- 常见问题
- 解决方案
- 诊断步骤

## 🔄 更新流程

### 1. 代码变更
```bash
# 1. 修改代码
# 2. 更新相关文档
# 3. 提交时包含文档变更
git commit -m "feat: 新功能 + 文档更新"
```

### 2. 文档审查
- 检查链接是否有效
- 验证代码示例可运行
- 确保格式一致

### 3. 版本说明
- 在 CHANGELOG.md 记录变更
- 更新迁移指南（如需要）
- 标记废弃的功能

---

## 🗑️ 文档清理

### 定期清理
- 每个版本发布前审查文档
- 删除过时的内容
- 合并重复的文档

### 标记废弃
```markdown
> ⚠️ **已废弃**：此功能已在 v2.0 中移除，请使用 [新功能](link)
```

### 归档历史
- 将历史文档移到 `docs/archive/`
- 保留重要的迁移指南
- 删除临时的开发文档
