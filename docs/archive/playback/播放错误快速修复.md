# 播放错误快速修复指南

## 🎯 问题

音箱播放音乐时显示"播放错误"。

## ✅ 解决方案（已修复）

### 核心问题

音乐平台的URL有防盗链保护，音箱无法直接访问。

### 修复内容

**1. 使用代理URL**

代码已修改为使用music_download的代理服务：
```python
# 原始URL: https://music.qq.com/xxx.mp3
# 代理URL: http://10.184.62.160:5050/main/proxy?url=https%3A%2F%2Fmusic.qq.com%2Fxxx.mp3
```

**2. 配置局域网IP**

`.env`文件已更新：
```env
MUSIC_API_BASE_URL=http://10.184.62.160:5050
```

**3. 添加停止延迟**

播放前等待0.5秒确保停止完成。

## 🔧 故障排查

### 如果还是播放失败

**1. 检查music_download服务**

确保music_download服务正在运行：
```bash
curl http://10.184.62.160:5050/main/proxy?url=https://www.baidu.com
```

应该返回百度首页内容。

**2. 检查局域网IP**

确认你的机器IP：
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

如果IP不是`10.184.62.160`，需要修改`.env`文件。

**3. 检查音箱和服务器网络**

确保音箱和运行后端的机器在同一局域网。

**4. 运行诊断脚本**

```bash
python test/music/diagnose_playback.py
```

应该看到：
```
✅ 配置检查
✅ 代理端点
```

## 📝 技术细节

### 为什么需要代理？

音乐平台的防盗链机制：
```
音箱直接访问 → 缺少Referer/User-Agent → 403 Forbidden → 播放失败
```

使用代理后：
```
音箱 → 代理服务器（添加headers）→ 音乐平台 → 返回流 → 音箱 → 成功
```

### 代理URL格式

```
http://{局域网IP}:{端口}/main/proxy?url={URL编码的原始URL}
```

例如：
```
原始: https://music.qq.com/song.mp3
代理: http://10.184.62.160:5050/main/proxy?url=https%3A%2F%2Fmusic.qq.com%2Fsong.mp3
```
