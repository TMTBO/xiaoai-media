### 当前项目的问题

**问题1：URL无法被音箱访问** ⭐⭐⭐⭐⭐

从音乐平台API（腾讯音乐、网易云等）获取的URL有**防盗链保护**：
- 需要特定的HTTP headers（Referer、User-Agent等）
- 音箱的播放器直接访问会被拒绝
- 导致播放失败

**问题2：使用localhost地址**

原配置：
```env
MUSIC_API_BASE_URL=http://localhost:5050
```

音箱是局域网中的独立设备，无法访问运行后端的机器上的localhost。

## ✅ 解决方案

### 修改1：使用代理URL

在`backend/src/xiaoai_media/api/routes/music.py`中修改`play_music`函数：

```python
# 获取原始URL
original_url = play_info["url"]

# 转换为代理URL，让音箱通过代理访问
from urllib.parse import quote
proxy_url = f"{config.MUSIC_API_BASE_URL}/main/proxy?url={quote(original_url)}"
url = proxy_url
```

**工作原理：**
1. 音箱访问：`http://10.184.62.160:5050/main/proxy?url=https%3A%2F%2Fmusic.qq.com%2Fxxx.mp3`
2. music_download代理服务器接收请求
3. 代理服务器添加必要的headers访问音乐平台
4. 将音乐流转发给音箱
5. ✅ 播放成功

### 修改2：更新配置为局域网IP

`.env`文件：
```env
MUSIC_API_BASE_URL=http://10.184.62.160:5050
```

**注意：** 请根据你的实际局域网IP修改，可通过以下命令查看：
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

### 修改3：添加停止延迟

在播放前确保停止完成：
```python
await client.player_stop(req.device_id)
await asyncio.sleep(0.5)  # 等待停止完成
await client.play_url(url, req.device_id, _type=1)
```

## 🎯 核心原理

**为什么需要代理？**

音乐平台的URL通常有防盗链保护：
```
直接访问 → 音乐平台检查headers → 缺少Referer/User-Agent → 拒绝访问 → 播放失败
```

使用代理后：
```
音箱 → 代理服务器 → 添加正确headers → 音乐平台 → 返回音乐流 → 代理转发 → 音箱 → 播放成功
```
