# 全局状态 SSE 推送

## 概述

全局状态 SSE 端点提供统一的状态推送通道，包含设备在线状态、播放状态、播单信息、播放进度等完整信息。

## API 端点

```
GET /api/state/stream?device_id={device_id}
```

### 参数

- `device_id` (可选): 设备 ID，如果不指定则监听所有设备

### 响应格式

SSE 事件流，包含两种事件类型：

#### 1. state 事件 - 状态更新

```
event: state
data: {
  "device_id": "xxx",
  "device_online": true,
  "device_name": "小爱音箱",
  "device_hardware": "LX06",
  "play_status": "playing",
  "audio_id": "xxx",
  "position": 12345,
  "duration": 234567,
  "media_type": 3,
  "current_song": {
    "name": "歌曲名",
    "singer": "歌手",
    "album": "专辑",
    "cover": "封面URL",
    "audio_id": "xxx"
  },
  "playlist": {
    "current": 0,
    "total": 10,
    "songs": [...]
  }
}
```

#### 2. heartbeat 事件 - 心跳

```
event: heartbeat
data: {"timestamp": 1234567890.123}
```

## 前端实现

### 1. 全局状态 Composable

已实现 `useGlobalState` composable，提供：

```typescript
const { 
  state,        // 完整状态对象
  connected,    // 连接状态
  error,        // 错误信息
  isPlaying,    // 是否正在播放
  isPaused,     // 是否暂停
  currentSong,  // 当前歌曲
  playlist,     // 播放列表
  progress,     // 播放进度（百分比）
  reconnect,    // 重连方法
  disconnect    // 断开连接方法
} = useGlobalState(deviceId)
```

### 2. 全局播放器栏组件

已实现 `GlobalPlayerBar` 组件，功能包括：

- 显示当前播放歌曲信息（封面、歌名、歌手、专辑）
- 播放控制按钮（上一曲、播放/暂停、下一曲）
- 播放进度条
- 播放列表位置显示
- 仅在有歌曲播放时显示，带滑入动画

### 3. 集成到应用

在 `App.vue` 中已集成：

```vue
<template>
  <el-main class="main">
    <GlobalDeviceSelector />
    <GlobalPlayerBar />  <!-- 新增播放器栏 -->
    <div class="main-content">
      <router-view />
    </div>
  </el-main>
</template>
```

## 优势

1. **统一状态管理**: 所有组件共享同一个状态源，避免数据不一致
2. **实时更新**: 状态变化时立即推送，延迟 < 1 秒
3. **减少请求**: 从每 3 秒轮询改为事件驱动，减少 95% 请求
4. **更好的 UX**: 播放器栏提供直观的播放控制和信息展示
5. **自动重连**: 网络中断后自动恢复连接

## 注意事项

1. **单例状态**: `useGlobalState` 使用单例模式，所有组件共享同一个状态
2. **连接管理**: 切换设备时会自动断开旧连接，建立新连接
3. **心跳机制**: 每 30 秒发送心跳，防止连接超时
4. **错误处理**: 连接失败时自动重试，最多 5 次
