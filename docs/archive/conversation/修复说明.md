# 对话监听功能修复说明

## 问题

在后台发送或音箱设备上输入的播放指令，没有通过本服务调用 `play_music` 获取音乐 URL。

## 根本原因

当用户直接对音箱说话（例如："小爱同学，播放周杰伦的晴天"），这个指令由小爱音箱的云服务直接处理，不会经过你的本地服务。因此无法：
- 使用你的音乐搜索服务
- 获取自定义的音乐 URL
- 控制播放列表

## 解决方案

参考 [xiaomusic](https://github.com/hanxi/xiaomusic) 项目的实现，通过**对话监听机制**拦截播放指令。

### 实现原理

xiaomusic 使用 `ConversationPoller` 持续轮询音箱的对话记录：

1. 每隔一定时间（默认 2 秒）请求音箱的对话历史
2. 检测新的对话记录（基于时间戳）
3. 解析对话内容，识别播放指令
4. 拦截播放指令，调用自己的音乐服务
5. 获取音乐 URL 并播放

## 实现细节

### 1. 新增模块

#### `backend/src/xiaoai_media/conversation.py`

**ConversationPoller 类** - 对话轮询器

核心功能：
- `start()` - 启动后台轮询任务
- `stop()` - 停止轮询
- `_poll_loop()` - 主轮询循环
- `_poll_conversations()` - 从所有设备获取对话
- `_process_conversations()` - 处理新对话记录

关键逻辑：
```python
# 记录每个设备的最后时间戳
self.last_timestamp: dict[str, int] = {}

# 只处理时间戳大于上次记录的对话
if timestamp > self.last_timestamp.get(device_id, 0):
    self.last_timestamp[device_id] = timestamp
    # 触发命令回调
    await self._command_callback(device_id, query)
```

#### `backend/src/xiaoai_media/command_handler.py`

**CommandHandler 类** - 命令处理器

核心功能：
- `handle_command()` - 处理检测到的指令
- `_parse_play_command()` - 解析播放指令
- `_handle_play_command()` - 执行播放流程
- `_search_music()` - 搜索音乐
- `_sync_playlist()` - 同步播放列表
- `_play_song()` - 播放歌曲

指令识别正则：
```python
r"^(?:播放|打开)(?:歌曲)?(.+)$"
```

支持格式：
- "播放周杰伦的晴天" → 提取 "周杰伦的晴天"
- "播放歌曲晴天" → 提取 "晴天"
- "打开周杰伦的歌" → 提取 "周杰伦的歌"

过滤逻辑：
```python
# 过滤掉控制指令
if content and not any(
    x in content for x in ["音量", "暂停", "继续", "停止", "下一首", "上一首"]
):
    return {"action": "play", "query": content}
```

## 完整流程示例

### 场景：用户对音箱说"播放周杰伦的晴天"

```
1. 用户: "小爱同学，播放周杰伦的晴天"
   ↓
2. 小爱音箱记录对话到云端
   ↓
3. ConversationPoller (2秒后轮询)
   - 调用 client.get_latest_ask(device_id)
   - 检测到新对话: "播放周杰伦的晴天"
   ↓
4. CommandHandler.handle_command()
   - 解析指令: _parse_play_command()
   - 识别为播放指令，提取 "周杰伦的晴天"
   ↓
5. CommandHandler._handle_play_command()
   - 调用 _search_music("周杰伦的晴天")
   - 请求: POST http://localhost:5050/api/v3/search
   - 返回: 20首搜索结果
   ↓
6. CommandHandler._sync_playlist()
   - 将搜索结果同步到 _playlists[device_id]
   - 创建播放列表（20首歌）
   ↓
7. CommandHandler._play_song(device_id, 0)
   - 调用 _get_play_url_with_fallback(第一首歌)
   - 尝试不同音质获取 URL
   - 获取到: http://music.example.com/xxx.mp3
   ↓
8. client.play_url(url, device_id)
   - 调用音箱播放 URL
   ↓
9. 音箱开始播放音乐 ✅
```

## 关键代码片段

### 对话轮询

```python
# conversation.py
async def _poll_conversations(self):
    async with XiaoAiClient() as client:
        devices = await client.list_devices()
        for device in devices:
            device_id = device.get("deviceID")
            conversations = await client.get_latest_ask(device_id)
            await self._process_conversations(device_id, conversations)
```

### 指令解析

```python
# command_handler.py
def _parse_play_command(self, query: str) -> dict | None:
    play_patterns = [r"^(?:播放|打开)(?:歌曲)?(.+)$"]
    for pattern in play_patterns:
        match = re.match(pattern, query)
        if match:
            content = match.group(1).strip()
            if content and not any(x in content for x in ["音量", "暂停", ...]):
                return {"action": "play", "query": content}
    return None
```

### 播放流程

```python
# command_handler.py
async def _handle_play_command(self, device_id: str, search_query: str):
    # 1. 搜索音乐
    search_results = await self._search_music(search_query)
    songs = search_results.get("data", {}).get("list", [])
    
    # 2. 同步播放列表
    await self._sync_playlist(device_id, songs)
    
    # 3. 播放第一首
    await self._play_song(device_id, 0)
```

## 故障排查

### 问题1：没有检测到对话

**可能原因：**
- 对话监听未启用
- 小米账号认证失败
- 设备 ID 不正确

**解决方法：**
1. 检查 `.env` 中 `ENABLE_CONVERSATION_POLLING=true`
2. 查看启动日志是否有 "对话轮询器已启动"
3. 测试 `/api/command/conversation` 接口是否能获取对话

### 问题2：检测到对话但没有播放

**可能原因：**
- 音乐服务不可用
- 搜索无结果
- URL 获取失败

**解决方法：**
1. 检查 `MUSIC_API_BASE_URL` 是否正确
2. 手动测试搜索接口：`curl http://localhost:5050/api/v3/search`
3. 查看详细日志中的错误信息

### 问题3：播放的歌曲不对

**原因：** 搜索结果的第一首可能不是你想要的

**解决方法：**
- 说话时更明确，包含歌手名
- 例如："播放周杰伦的晴天" 而不是 "播放晴天"

## 性能影响

- **CPU**: 轮询任务很轻量，几乎无影响
- **内存**: 每个设备存储一个时间戳，可忽略
- **网络**: 每个轮询周期请求一次小米 API（每设备）
- **建议**: 如果有多个设备，可以适当增加轮询间隔
