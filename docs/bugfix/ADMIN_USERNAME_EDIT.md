# Admin 用户名修改功能

## 问题描述

之前的版本中，admin 用户的用户名无法修改。在用户管理界面编辑用户时，用户名字段被禁用。

## 解决方案

### 后端修改

后端代码已经支持管理员修改用户名，无需修改。`backend/src/xiaoai_media/api/routes/auth.py` 中的 `update_user` 接口允许管理员修改任何用户的用户名。

### 前端修改

1. **用户管理界面** (`frontend/src/views/UserManagement.vue`)
   - 移除了编辑模式下用户名字段的禁用状态
   - 添加了"新用户名"字段，用于输入要修改的用户名
   - 在编辑对话框中，显示当前用户名（禁用状态）和新用户名输入框
   - 添加了自动重新登录逻辑：当用户修改自己的用户名后，会自动跳转到登录页

2. **API 接口** (`frontend/src/api/auth.ts`)
   - 更新了 `UpdateUserRequest` 接口，添加 `new_username` 字段

### 测试

添加了两个新的测试用例：
- `test_update_admin_username`: 验证可以修改 admin 用户名
- `test_update_username_duplicate`: 验证不能修改为已存在的用户名

## 使用方法

1. 以管理员身份登录
2. 进入"用户管理"页面
3. 点击要编辑的用户（包括 admin）的"编辑"按钮
4. 在"新用户名"字段中输入新的用户名
5. 点击"确定"保存

**注意**: 如果修改的是当前登录用户的用户名，系统会自动跳转到登录页，需要使用新用户名重新登录。

## 技术细节

- 用户名修改通过 `PUT /api/users/{username}` 接口实现
- 请求参数中的 `new_username` 字段用于指定新用户名
- 后端会检查新用户名是否已存在，如果存在则返回错误
- 修改用户名后，用户的密码、角色等信息保持不变
- 如果修改的是当前登录用户的用户名，前端会清除本地存储的认证信息并跳转到登录页
