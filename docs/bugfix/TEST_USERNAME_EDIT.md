# 用户名编辑功能测试步骤

## 前提条件

1. 确保前端开发服务器正在运行
2. 确保后端服务正在运行
3. 已经以管理员身份登录

## 测试步骤

### 步骤 1：重启前端服务

```bash
# 停止当前运行的前端服务 (Ctrl+C)
# 然后重新启动
cd frontend
npm run dev
```

### 步骤 2：清除浏览器缓存

1. 打开浏览器开发者工具 (F12 或 Cmd+Option+I)
2. 右键点击浏览器刷新按钮
3. 选择"清空缓存并硬性重新加载"

或者直接按：
- Mac: `Cmd + Shift + R`
- Windows/Linux: `Ctrl + Shift + F5`

### 步骤 3：访问用户管理页面

1. 在浏览器中访问：`http://localhost:5173/users` (或你的前端地址)
2. 确认页面加载成功，显示用户列表

### 步骤 4：编辑用户

1. 点击任意用户（包括 admin）的"编辑"按钮
2. 应该弹出编辑对话框

### 步骤 5：检查对话框内容

编辑对话框应该包含以下字段：

```
┌─────────────────────────────────────┐
│  编辑用户                      ✕    │
├─────────────────────────────────────┤
│                                     │
│  当前用户名  [admin        ] (灰色) │
│                                     │
│  新用户名    [              ] ✕     │
│              留空则不修改用户名      │
│                                     │
│  密码        [              ]       │
│              留空则不修改密码        │
│                                     │
│  角色        [管理员 ▼]             │
│                                     │
├─────────────────────────────────────┤
│              [取消]  [确定]         │
└─────────────────────────────────────┘
```

### 步骤 6：输入新用户名

1. 在"新用户名"字段中输入：`superadmin`
2. 确认可以正常输入
3. 点击"确定"按钮

### 步骤 7：验证结果

**如果修改的是当前登录用户：**
- 应该显示提示："用户名修改成功，请重新登录"
- 1.5 秒后自动跳转到登录页
- 使用新用户名 `superadmin` 和原密码登录

**如果修改的是其他用户：**
- 应该显示提示："用户更新成功"
- 对话框关闭
- 用户列表刷新，显示新用户名

## 调试检查

### 检查 1：查看网络请求

1. 打开开发者工具 (F12)
2. 切换到 Network 标签页
3. 点击"编辑"按钮
4. 输入新用户名并提交
5. 查找 PUT 请求到 `/api/users/admin`
6. 点击该请求，查看 Payload：

```json
{
  "new_username": "superadmin",
  "role": "admin"
}
```

### 检查 2：查看控制台错误

1. 打开开发者工具 (F12)
2. 切换到 Console 标签页
3. 查看是否有红色错误信息
4. 如果有错误，记录错误信息

### 检查 3：验证代码更新

在浏览器控制台执行：

```javascript
// 检查 Vue 组件是否包含新字段
console.log('检查 UserManagement 组件...');
```

或者直接查看源代码：
1. 在开发者工具中，切换到 Sources 标签页
2. 找到 `UserManagement.vue`
3. 搜索 "新用户名"
4. 确认代码已更新

## 预期行为

### 创建用户时
- 只显示"用户名"字段（必填）
- 不显示"当前用户名"和"新用户名"

### 编辑用户时
- 显示"当前用户名"（禁用，灰色）
- 显示"新用户名"（可选，可清除）
- 密码字段可选
- 角色可选择

## 常见问题排查

### 问题：看不到"新用户名"字段

**可能原因：**
1. 浏览器缓存未清除
2. 前端服务未重启
3. 代码未正确保存

**解决方法：**
```bash
# 1. 停止前端服务
# 2. 清除 node_modules/.vite 缓存
rm -rf frontend/node_modules/.vite

# 3. 重新启动
cd frontend
npm run dev

# 4. 硬刷新浏览器 (Cmd+Shift+R)
```

### 问题：字段显示但无法输入

**检查：**
1. 是否有 JavaScript 错误？
2. Element Plus 是否正常加载？
3. 输入框是否被 CSS 覆盖？

**调试：**
```javascript
// 在控制台执行
document.querySelector('input[placeholder="留空则不修改用户名"]')
```

### 问题：提交时报错

**检查后端日志：**
```bash
# Docker 方式
docker-compose logs -f backend

# 直接运行方式
cd backend
python run.py
```

**检查请求参数：**
- 打开 Network 标签页
- 查看 PUT 请求的 Request Payload
- 确认包含 `new_username` 字段
