## 快速开始

### 1. 部署服务

使用 Docker Compose 快速部署：

```bash
# 1. 创建配置文件
mkdir -p ./data
cp user_config_template.py ./data/user_config.py
cp music_provider_template.py ./data/music_provider.py

# 2. 编辑配置
vim ./data/user_config.py
# 填入你的小米账号和密码

# 3. 启动服务
docker-compose up -d

# 4. 访问管理界面
# 浏览器打开 http://localhost:8000
```

### 2. 首次配置

1. 打开管理界面
2. 进入"设备列表"页面
3. 查看你的小爱音箱设备
4. 记下设备 ID，用于后续操作

## 设备管理

### 查看设备列表

在管理界面的"设备列表"页面，可以看到：
- 设备名称
- 设备 ID
- 在线状态
- 硬件型号
- IP 地址

### 全局设备选择器

页面顶部有全局设备选择器，选择后会在所有页面生效：
- 实时显示设备状态（在线/离线）
- 实时显示播放状态（播放中/暂停/停止）
- 快速停止播放按钮
- 显示设备详细信息

## 播放列表

### 创建播放列表

1. 进入"播单管理"页面
2. 点击"创建播单"按钮
3. 填写播单信息：
   - 播单名称（必填）
   - 播单类型（音乐/有声书/播客/其他）
   - 描述（可选）
   - 语音关键词（用于语音播放）

示例：
```
名称：我的音乐
类型：音乐
语音关键词：音乐、歌曲
```

### 添加播放项

#### 方式 1：手动添加

1. 在播单列表中点击"编辑"
2. 点击"添加项"按钮
3. 填写歌曲信息：
   - 标题（必填）
   - 艺术家（可选）
   - 专辑（可选）
   - Audio ID（必填）
   - URL（可选）

#### 方式 2：批量导入

1. 点击"批量导入"按钮
2. 选择音频文件所在目录
3. 系统自动扫描并导入所有音频文件
4. 支持自然排序（如：第1章、第2章、第10章）

详见：[批量导入指南](playlist/README_BATCH_IMPORT.md)

### 语音播放

对小爱音箱说：
- "小爱，播放音乐播单"
- "小爱，播放有声书"
- "小爱，播放我的播客"

系统会根据语音关键词自动匹配播放列表。

### 语音播放

## 定时任务

### 创建定时播放

1. 进入"定时任务"页面
2. 点击"创建任务"按钮
3. 选择任务类型：
   - 播放音乐
   - 播放播放列表
   - 提醒
   - 执行指令

4. 设置触发时间：
   - Cron 表达式（周期性任务）
   - 具体时间（一次性任务）
   - 延迟分钟数（延迟任务）

5. 填写任务参数

### Cron 表达式示例

```
0 7 * * *       # 每天早上 7 点
30 20 * * 1-5   # 周一到周五晚上 8:30
0 */2 * * *     # 每 2 小时
0 12 * * 0      # 每周日中午 12 点
```

### 使用场景

- 每天早上 7 点播放起床音乐
- 工作日晚上 8 点播放放松音乐
- 每 2 小时提醒喝水
- 30 分钟后停止播放

详见：[定时任务指南](scheduler/README.md)

## 语音命令

### 执行自定义命令

1. 进入"语音指令"页面
2. 输入要执行的命令
3. 选择目标设备
4. 点击"执行"按钮

### 支持的命令

- 播放控制：播放、暂停、停止、下一首、上一首
- 音量控制：音量加、音量减、设置音量
- 音乐播放：播放周杰伦的歌、播放热歌榜
- 播放列表：播放音乐播单、播放有声书
- 其他：查询天气、设置闹钟等

### 智能解析

系统会自动解析语音命令：
- 识别播放列表关键词
- 识别排行榜名称
- 识别音乐搜索关键词
- 执行相应操作

## 对话监听

### 启用对话监听

1. 在配置文件中启用：
```python
# user_config.py
ENABLE_CONVERSATION_MONITOR = True
WAKE_WORDS = ["小爱", "播放"]
ENABLE_WAKE_WORD_FILTER = True
```

2. 重启服务

### 工作原理

1. 监听音箱的对话历史
2. 识别包含唤醒词的指令
3. 自动拦截播放指令
4. 执行自定义播放逻辑

### 使用场景

- 自动播放自定义播放列表
- 拦截默认音乐播放，使用自己的音乐源
- 记录对话历史

详见：[对话监听指南](conversation/README.md)

## TTS 播报

### 使用 TTS

1. 进入"TTS 朗读"页面
2. 输入要播报的文字
3. 选择目标设备
4. 点击"播报"按钮

### API 调用

```bash
curl -X POST http://localhost:8000/api/tts \
  -H "Content-Type: application/json" \
  -d '{
    "text": "您好，现在是下午3点",
    "device_id": "your_device_id"
  }'
```

### 使用场景

- 定时提醒
- 语音通知
- 智能家居联动

## 常见问题

### 1. 无法连接设备

检查：
- 小米账号和密码是否正确
- 设备是否在线
- 网络是否正常

### 2. 播放失败

检查：
- 音乐 API 是否正常
- `SERVER_BASE_URL` 是否配置为局域网 IP
- 音频 URL 是否可访问

### 3. 语音命令不响应

检查：
- 对话监听是否启用
- 唤醒词配置是否正确
- 查看对话历史是否有记录

### 4. 定时任务不执行

检查：
- 任务是否启用
- Cron 表达式是否正确
- 查看应用日志

## 📚 进阶使用

### 自定义音频源

在 `user_config.py` 中实现 `get_audio_url()` 函数：

```python
def get_audio_url(audio_id: str, custom_params: dict = None) -> str:
    """自定义音频 URL 获取逻辑"""
    # 从你的音乐服务获取 URL
    return f"http://your-music-api.com/song/{audio_id}"
```

### 自定义命令处理

```python
def should_handle_command(query: str) -> bool:
    """自定义命令过滤逻辑"""
    if not any(word in query for word in WAKE_WORDS):
        return False
    return "播放" in query or "音乐" in query

def preprocess_command(query: str) -> str:
    """自定义命令预处理逻辑"""
    for wake_word in WAKE_WORDS:
        query = query.replace(wake_word, "")
    return query.strip()
```
