# 定时任务前端测试指南

## 启动前端开发服务器

```bash
cd frontend
npm install  # 如果还没安装依赖
npm run dev
```

访问：http://localhost:5173

## 测试步骤

### 1. 访问定时任务页面

在左侧菜单中点击"定时任务"，应该能看到：
- 页面标题"定时任务管理"
- 创建任务按钮
- 任务类型筛选器
- 快捷操作面板（三个卡片）
- Cron 表达式帮助

### 2. 测试快速提醒

在"快速提醒"卡片中：
1. 输入提醒内容：`该喝水了`
2. 设置延迟分钟数：`1`（测试用，1分钟后）
3. 点击"创建提醒"按钮
4. 应该看到成功提示
5. 任务列表中应该出现新创建的任务

### 3. 测试快速播放音乐

在"定时播放音乐"卡片中：
1. 输入歌曲名称：`晴天`
2. 输入歌手：`周杰伦`
3. 输入 Cron 表达式：`0 7 * * *`（每天早上7点）
4. 点击"创建任务"按钮
5. 应该看到成功提示
6. 任务列表中应该出现新任务，显示下次执行时间

### 4. 测试快速播放播放列表

前提：需要先创建一个播放列表

在"定时播放播放列表"卡片中：
1. 从下拉框选择一个播放列表
2. 输入 Cron 表达式：`0 20 * * 1-5`（工作日晚上8点）
3. 点击"创建任务"按钮
4. 应该看到成功提示

### 5. 测试任务列表

任务列表应该显示：
- 任务名称
- 任务类型标签（不同颜色）
- 触发方式（Cron 表达式或"一次性"）
- 下次执行时间
- 任务参数
- 启用/禁用开关
- 编辑和删除按钮

### 6. 测试启用/禁用任务

1. 点击任务的启用/禁用开关
2. 应该看到状态切换成功的提示
3. 禁用的任务不会显示"下次执行时间"

### 7. 测试编辑任务

1. 点击任务的"编辑"按钮
2. 应该弹出编辑对话框，显示当前任务信息
3. 修改任务名称或其他参数
4. 点击"更新"按钮
5. 应该看到更新成功的提示
6. 任务列表中的信息应该更新

### 8. 测试删除任务

1. 点击任务的"删除"按钮
2. 应该弹出确认对话框
3. 点击"删除"确认
4. 应该看到删除成功的提示
5. 任务从列表中消失

### 9. 测试创建任务对话框

点击页面顶部的"创建任务"按钮：

#### 创建播放音乐任务
1. 选择任务类型：播放音乐
2. 输入任务名称：`早安音乐`
3. 选择触发方式：周期性（Cron）
4. 输入 Cron 表达式：`0 7 * * *`
5. 输入歌曲名称：`晴天`
6. 输入歌手：`周杰伦`
7. 启用开关：开启
8. 点击"创建"按钮

#### 创建播放播放列表任务
1. 选择任务类型：播放播放列表
2. 输入任务名称：`晚间放松`
3. 选择触发方式：周期性（Cron）
4. 输入 Cron 表达式：`0 20 * * *`
5. 选择播放列表
6. 点击"创建"按钮

#### 创建一次性提醒任务
1. 选择任务类型：提醒
2. 输入任务名称：`下午开会提醒`
3. 选择触发方式：一次性
4. 选择执行时间：选择未来的某个时间
5. 输入提醒内容：`该开会了`
6. 点击"创建"按钮

### 10. 测试任务类型筛选

1. 在筛选器中选择"播放音乐"
2. 任务列表应该只显示播放音乐类型的任务
3. 选择"全部任务"
4. 应该显示所有任务

### 11. 测试表单验证

尝试创建任务但不填写必填字段：
1. 点击"创建任务"
2. 不填写任何字段，直接点击"创建"
3. 应该看到表单验证错误提示

### 12. 测试响应式布局

1. 调整浏览器窗口大小
2. 快捷操作卡片应该自适应布局
3. 在移动端视图下，任务项的操作按钮应该换行显示

## 验证后端集成

### 检查 API 调用

打开浏览器开发者工具（F12），切换到 Network 标签：

1. 创建任务时，应该看到 POST 请求到 `/api/scheduler/tasks/cron` 或 `/api/scheduler/tasks/date`
2. 加载任务列表时，应该看到 GET 请求到 `/api/scheduler/tasks`
3. 更新任务时，应该看到 PUT 请求到 `/api/scheduler/tasks/{id}`
4. 删除任务时，应该看到 DELETE 请求到 `/api/scheduler/tasks/{id}`

### 检查任务执行

1. 创建一个1分钟后的提醒任务
2. 等待1分钟
3. 小爱音箱应该播报提醒内容
4. 任务应该从列表中自动消失（一次性任务执行后会被删除）

## 常见问题

### 1. 页面空白或报错

检查：
- 后端服务是否正常运行（http://localhost:8000）
- 浏览器控制台是否有错误信息
- Network 标签中 API 请求是否成功

### 2. 播放列表下拉框为空

检查：
- 是否已经创建了播放列表
- 访问 http://localhost:8000/api/playlists 查看是否有数据

### 3. 任务创建成功但不执行

检查：
- 后端日志中是否有错误信息
- 任务是否已启用
- Cron 表达式是否正确
- 执行时间是否已到

### 4. 时间显示不正确

检查：
- 浏览器时区设置
- 后端时区配置（应该是 Asia/Shanghai）
