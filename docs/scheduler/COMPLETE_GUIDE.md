# 定时任务功能完整指南

## 功能概述

定时任务功能已完整实现，包括后端 API 和前端管理界面。用户可以通过 Web 界面轻松创建和管理各种定时任务。

## 已实现的功能

### ✅ 后端功能

1. **任务调度系统**
   - 基于 APScheduler 的异步调度
   - 支持 Cron 表达式（周期性任务）
   - 支持一次性定时任务
   - 支持延迟任务

2. **任务类型**
   - 定时播放音乐
   - 定时播放播放列表
   - 定时提醒

3. **数据持久化**
   - 任务配置保存到 `~/.xiaoai_media/scheduler/tasks.json`
   - 应用重启自动恢复任务
   - 过期任务自动清理

4. **完整的 REST API**
   - 创建任务（Cron/一次性/延迟）
   - 查询任务（列表/详情）
   - 更新任务
   - 删除任务
   - 快捷操作 API

### ✅ 前端功能

1. **任务列表管理**
   - 展示所有定时任务
   - 按任务类型筛选
   - 显示任务状态和下次执行时间
   - 快速启用/禁用任务
   - 编辑和删除任务

2. **创建任务对话框**
   - 选择任务类型
   - 配置触发方式（Cron 或一次性）
   - 输入任务参数
   - 表单验证

3. **快捷操作面板**
   - 快速创建提醒
   - 快速创建定时播放音乐
   - 快速创建定时播放播放列表
   - Cron 表达式帮助

4. **用户体验**
   - 响应式设计，支持移动端
   - 实时反馈（成功/错误提示）
   - 加载状态指示
   - 友好的空状态提示

## 快速开始

### 1. 启动后端

```bash
cd backend
pip install -e .
python run.py
```

后端将在 http://localhost:8000 启动

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端将在 http://localhost:5173 启动

### 3. 访问管理界面

打开浏览器访问 http://localhost:5173，在左侧菜单点击"定时任务"。

## 使用示例

### 场景 1：每天早上 7 点播放起床音乐

1. 在快捷操作面板的"定时播放音乐"卡片中：
   - 歌曲名称：`晴天`
   - 歌手：`周杰伦`
   - Cron 表达式：`0 7 * * *`
2. 点击"创建任务"
3. 任务创建成功，每天早上 7 点会自动播放

### 场景 2：工作日晚上 8 点播放放松音乐

1. 先在"播单管理"页面创建一个播放列表
2. 在"定时播放播放列表"卡片中：
   - 选择刚创建的播放列表
   - Cron 表达式：`0 20 * * 1-5`
3. 点击"创建任务"
4. 周一到周五晚上 8 点会自动播放该播放列表

### 场景 3：10 分钟后提醒喝水

1. 在"快速提醒"卡片中：
   - 提醒内容：`该喝水了`
   - 延迟分钟数：`10`
2. 点击"创建提醒"
3. 10 分钟后小爱音箱会语音播报提醒

### 场景 4：每 2 小时提醒休息

1. 点击页面顶部的"创建任务"按钮
2. 在对话框中：
   - 任务类型：提醒
   - 任务名称：`休息提醒`
   - 触发方式：周期性（Cron）
   - Cron 表达式：`0 */2 * * *`
   - 提醒内容：`该休息一下了`
3. 点击"创建"
4. 每 2 小时会收到提醒

## Cron 表达式说明

格式：`分 时 日 月 周`

### 常用示例

| 表达式 | 说明 |
|--------|------|
| `0 7 * * *` | 每天早上 7 点 |
| `30 20 * * *` | 每天晚上 8 点 30 分 |
| `0 */2 * * *` | 每 2 小时 |
| `0 9-17 * * *` | 每天 9 点到 17 点，每小时 |
| `0 7 * * 1-5` | 周一到周五早上 7 点 |
| `0 12 * * 0` | 每周日中午 12 点 |
| `0 0 1 * *` | 每月 1 号凌晨 |
| `*/30 * * * *` | 每 30 分钟 |

### 字段说明

- **分**：0-59
- **时**：0-23
- **日**：1-31
- **月**：1-12
- **周**：0-6（0 = 周日）

### 特殊字符

- `*`：任意值
- `,`：列举多个值，如 `1,3,5`
- `-`：范围，如 `1-5`
- `/`：步长，如 `*/2` 表示每 2 个单位

## API 端点

### 基础管理

```
POST   /api/scheduler/tasks/cron      创建 Cron 任务
POST   /api/scheduler/tasks/date      创建一次性任务
POST   /api/scheduler/tasks/delay     创建延迟任务
GET    /api/scheduler/tasks           列出所有任务
GET    /api/scheduler/tasks/{id}      获取任务详情
PUT    /api/scheduler/tasks/{id}      更新任务
DELETE /api/scheduler/tasks/{id}      删除任务
```

### 快捷操作

```
POST   /api/scheduler/quick/reminder        快速提醒
POST   /api/scheduler/quick/play-music      快速播放音乐
POST   /api/scheduler/quick/play-playlist   快速播放播放列表
```

详细的 API 文档请查看：[API 文档](./README.md)

## 数据流程

```
用户操作（前端）
    ↓
API 请求
    ↓
后端路由（scheduler.py）
    ↓
调度服务（scheduler_service.py）
    ↓
持久化（tasks.json）+ 添加到调度器
    ↓
定时触发
    ↓
任务执行器（scheduler_executor.py）
    ↓
音乐服务/播放列表服务/TTS 服务
    ↓
小爱音箱执行
```

## 故障排查

### 前端问题

1. **页面空白**
   - 检查后端是否运行
   - 检查浏览器控制台错误
   - 检查 Network 标签中的 API 请求

2. **任务列表不显示**
   - 检查 API 请求是否成功
   - 检查后端日志
   - 尝试刷新页面

3. **创建任务失败**
   - 检查表单验证错误
   - 检查 API 响应错误信息
   - 检查后端日志

### 后端问题

1. **任务不执行**
   - 检查任务是否启用
   - 检查 Cron 表达式是否正确
   - 查看后端日志中的错误信息

2. **任务执行失败**
   - 检查音乐服务是否可用
   - 检查播放列表是否存在
   - 查看详细错误日志

3. **任务丢失**
   - 检查持久化文件：`~/.xiaoai_media/scheduler/tasks.json`
   - 检查文件权限
   - 查看应用启动日志
