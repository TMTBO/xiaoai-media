# 播放器栏显示修复

## 问题

设备正在播放音乐，但播放器栏不显示。

## 原因分析

1. **播放列表为空**: 设备通过语音命令或其他方式播放，不是通过我们的播放列表系统
2. **歌曲信息缺失**: 小米音箱 API 的 `play_song_detail` 只返回 `audio_id`, `position`, `duration`，不包含歌名、歌手等信息

## 解决方案

### 后端修改

在 `state.py` 中添加回退逻辑：

1. **优先从播放列表获取**: 如果有播放列表，从中获取完整的歌曲信息
2. **回退到设备状态**: 如果没有播放列表，从设备的 `play_song_detail` 获取基本信息
3. **提供基本信息**: 即使没有歌名，也返回 `audio_id` 等基本信息

### 前端修改

在 `GlobalPlayerBar.vue` 中：

1. **调整显示条件**: 
   - 有完整歌曲信息时显示
   - 或正在播放音乐（`media_type=3` 且 `duration > 0`）时也显示

2. **显示占位信息**:
   - 歌名: "正在播放"
   - 歌手: "ID: xxx..." (显示部分 audio_id)
   - 仍然显示进度条和控制按钮

## 使用场景

### 场景 1: 通过播放列表播放（完整信息）

```json
{
  "current_song": {
    "name": "七里香",
    "singer": "周杰伦",
    "album": "七里香",
    "cover": "https://...",
    "audio_id": "xxx"
  },
  "playlist": {
    "current": 0,
    "total": 10
  }
}
```

播放器栏显示：
- 封面图片
- 歌名: 七里香
- 歌手: 周杰伦 · 专辑: 七里香
- 播放列表: 1/10

### 场景 2: 通过语音命令播放（基本信息）

```json
{
  "current_song": null,
  "playlist": null,
  "audio_id": "1774606909778686",
  "duration": 271107,
  "media_type": 3
}
```

播放器栏显示：
- 默认图标
- 歌名: 正在播放
- 歌手: ID: 177460690977...
- 进度条正常显示
- 播放列表: - (不显示)

## 重启服务

修改后需要重启后端服务：

```bash
# 停止当前服务 (Ctrl+C)
cd backend
python run.py
```

前端会自动热更新，无需重启。

## 验证

1. 通过语音命令播放音乐: "小爱同学，播放周杰伦的歌"
2. 刷新浏览器页面
3. 播放器栏应该显示，即使没有完整歌曲信息
4. 显示 "正在播放" 和 audio_id
5. 进度条正常更新

## 注意事项

1. **上一曲/下一曲按钮**: 当没有播放列表时会被禁用（因为不知道下一首是什么）
2. **播放/暂停按钮**: 始终可用，可以控制当前播放
3. **完整信息**: 建议通过播放列表系统播放，以获得完整的歌曲信息和列表控制

## 推荐使用方式

为了获得最佳体验，建议：

1. 使用"音乐搜索"功能搜索并播放
2. 使用"播单管理"加载播放列表
3. 这样可以获得完整的歌曲信息和列表控制功能
