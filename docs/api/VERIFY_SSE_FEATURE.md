# SSE 功能验证指南

## 修复内容

已修复 App.vue 中缺少的图标导入，页面应该可以正常显示了。

## 启动步骤

### 1. 启动后端

```bash
cd backend
python run.py
```

等待看到：
```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 2. 启动前端

```bash
cd frontend
npm run dev
```

等待看到：
```
VITE ready in xxx ms
Local: http://localhost:5173/
```

### 3. 打开浏览器

访问 `http://localhost:5173`

**重要**: 如果页面仍然空白，请：
1. 硬刷新页面 (Ctrl+Shift+R 或 Cmd+Shift+R)
2. 或使用无痕模式
3. 或清除浏览器缓存

## 功能验证

### 基础功能

1. ✅ 页面正常显示（左侧菜单 + 顶部设备选择器）
2. ✅ 可以选择设备
3. ✅ 设备选择器显示在线状态和播放状态

### SSE 连接

1. 打开浏览器开发者工具 (F12)
2. 切换到 Console 标签
3. 选择一个设备
4. 应该看到日志：`Global state SSE connected: xxx`

### 播放器栏

1. 进入"音乐搜索"页面
2. 搜索歌曲（例如："周杰伦"）
3. 点击播放按钮
4. **播放器栏应该自动出现**在设备选择器下方
5. 显示内容：
   - 歌曲封面（如果有）
   - 歌名、歌手、专辑
   - 播放/暂停按钮
   - 上一曲/下一曲按钮
   - 实时进度条
   - 播放列表位置 (1/10)

### 播放控制

1. 点击"暂停"按钮 → 音乐暂停，按钮变为"播放"
2. 点击"播放"按钮 → 音乐继续，按钮变为"暂停"
3. 点击"下一曲" → 切换到下一首歌
4. 点击"上一曲" → 切换到上一首歌
5. 观察进度条实时更新

### 状态同步

1. 在设备选择器中，播放状态应该显示"播放中"或"已暂停"
2. 状态变化应该实时反映（< 1 秒延迟）
3. 切换设备，播放器栏应该更新为新设备的状态

## 性能验证

### 查看请求数量

1. 打开开发者工具 Network 标签
2. 筛选 `status`
3. 播放音乐 1 分钟
4. 应该只看到 1 个 `state/stream` 连接（EventStream 类型）
5. **不应该**看到多个 `/api/music/status` 请求

### 查看后端日志

```bash
tail -f backend/logs/app.log | grep "SSE\|通知.*订阅者"
```

应该看到：
```
INFO: SSE 全局状态客户端已连接: device_id=xxx
DEBUG: 通知 1 个订阅者设备 xxx 状态变化
```

## 常见问题

### Q: 页面仍然空白？

A: 
1. 检查浏览器控制台是否有错误
2. 确认前端服务正在运行 (http://localhost:5173)
3. 尝试访问 http://localhost:5173/devices 直接进入设备页面
4. 清除浏览器缓存并硬刷新

### Q: 播放器栏不显示？

A:
1. 确认已选择设备
2. 确认设备正在播放音乐（不是 TTS 或其他内容）
3. 打开 Console 查看 `currentSong` 是否有值
4. 检查 SSE 是否连接成功

### Q: 按钮点击无反应？

A:
1. 检查 Console 是否有错误
2. 确认后端 API 正常（/api/music/next, /api/music/pause 等）
3. 查看 Network 标签确认请求已发送

### Q: 状态更新延迟？

A:
1. 正常延迟应该 < 1 秒
2. 检查 Network 标签中的 EventStream 是否有事件推送
3. 查看后端日志中的轮询间隔设置（默认 3 秒）
