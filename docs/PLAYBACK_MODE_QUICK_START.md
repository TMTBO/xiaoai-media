# 播放模式快速开始

## 什么是播放模式？

播放模式决定了系统如何监控播放状态并自动播放下一曲。有两种模式可选：

### 轮询模式（Monitor）- 默认

- ✅ 兼容性好，适用于所有场景
- ✅ 能检测各种异常情况
- ❌ 频繁调用 API，消耗资源
- ❌ 响应有延迟

### 定时器模式（Controller）- 推荐

- ✅ 高效节能，性能提升 30 倍
- ✅ 响应及时，切换准确
- ❌ 依赖准确的时长信息
- ❌ 无法检测异常情况

## 快速配置

### 方法一：前端配置（推荐）

1. 打开浏览器访问 `http://localhost:8000`
2. 点击左侧菜单"配置管理"
3. 找到"播放监控"部分
4. 选择监控模式：
   - **轮询模式**：定期检查播放状态（兼容性好，但较耗性能）
   - **定时器模式**：根据音频时长设置定时器（高效，但依赖准确的时长信息）
5. 点击"保存配置"

配置会立即生效，无需重启服务！

### 方法二：配置文件

编辑 `user_config.py`：

```python
# 启用播放监控
ENABLE_PLAYBACK_MONITOR = True

# 选择播放模式
PLAYBACK_MODE = "controller"  # 或 "monitor"
```

保存后，在前端配置页面点击"保存配置"以重新加载。

## 使用建议

### 推荐使用定时器模式的场景

- ✅ 播放本地音频文件（时长准确）
- ✅ 播放列表较长，追求性能
- ✅ 设备性能有限，需要节省资源

### 推荐使用轮询模式的场景

- ✅ 播放在线音频（时长可能不准确）
- ✅ 需要检测手动停止等异常
- ✅ 追求稳定性和兼容性

### 1. 查看日志

启动服务后，查看日志输出：


### 2. 测试播放

1. 创建一个播放列表
2. 开始播放
3. 观察日志：
   - 定时器模式：会显示"设置播放定时器"
   - 轮询模式：会显示"播放监控器已自动启动"

### 3. 观察性能

播放一首 3 分钟的歌曲：

- **定时器模式**：日志中只有 2-3 条播放相关记录
- **轮询模式**：日志中有 60+ 条状态检查记录

## 常见问题

### Q: 定时器模式下，歌曲没有自动切换？

A: 可能的原因：
1. 音频时长信息不准确 → 切换回轮询模式
2. 缓冲时间设置不当 → 调整缓冲时间（默认 1 秒）
3. 播放列表信息丢失 → 检查日志确认

### Q: 轮询模式下，切换延迟太大？

A: 可以调整轮询间隔：
```python
PLAYBACK_MONITOR_INTERVAL = 2.0  # 减小到 2 秒
```

### Q: 如何切换模式？

A: 两种方法：
1. 前端配置页面修改并保存
2. 修改 `user_config.py` 后在前端保存配置

配置会自动重新加载，无需重启！

### Q: 两种模式可以同时使用吗？

A: 不可以，同一时间只能使用一种模式。但可以随时切换。
