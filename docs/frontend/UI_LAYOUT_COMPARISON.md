## 布局切换逻辑

### 实现方式

```vue
<!-- App.vue -->
<template>
  <!-- 登录页面：全屏显示 -->
  <div v-if="isLoginPage" class="login-layout">
    <router-view />
  </div>

  <!-- 主应用：带侧边栏布局 -->
  <el-container v-else class="app-layout">
    <el-aside><!-- 侧边栏 --></el-aside>
    <el-main><!-- 主内容 --></el-main>
  </el-container>
</template>

<script setup>
const route = useRoute()
const isLoginPage = computed(() => route.path === '/login')
</script>
```

### 切换时机

1. **访问 `/login` 路由**
   - 显示全屏登录页面
   - 隐藏侧边栏和其他 UI 元素

2. **访问其他路由**
   - 显示完整应用布局
   - 包含侧边栏、设备选择器、播放器栏等

3. **路由守卫自动处理**
   - 未登录访问其他页面 → 自动跳转到 `/login`
   - 已登录访问 `/login` → 自动跳转到 `/devices`

## 响应式设计

### 桌面端
- 登录卡片宽度: 400px
- 侧边栏宽度: 200px
- 主内容区域: 自适应

### 移动端
- 登录卡片: 自适应宽度，最大 400px
- 侧边栏: 可考虑折叠或抽屉式
- 主内容: 全宽显示

## 用户体验优势

### 登录页面全屏显示的优点

1. **更专注** - 用户注意力集中在登录表单上
2. **更美观** - 全屏渐变背景视觉效果更好
3. **更清晰** - 明确区分登录和已登录状态
4. **更现代** - 符合现代 Web 应用设计规范
5. **更友好** - 移动端体验更好

### 已登录布局的优点

1. **功能完整** - 所有功能一目了然
2. **导航便捷** - 左侧菜单快速切换
3. **信息丰富** - 设备状态、播放信息实时显示
4. **操作高效** - 常用功能快速访问

## 技术实现细节

### CSS 样式

```css
/* 登录布局 - 全屏 */
.login-layout {
  width: 100%;
  height: 100vh;
}

/* 应用布局 - 侧边栏 + 主内容 */
.app-layout {
  height: 100vh;
}

.aside {
  width: 200px;
  background: #1d2d44;
  display: flex;
  flex-direction: column;
}

.main {
  flex: 1;
  background: #f5f7fa;
}
```

### 条件渲染

- 使用 `v-if` 和 `v-else` 实现条件渲染
- 根据 `route.path` 判断当前路由
- 动态切换布局组件

## 总结

通过条件渲染实现了两种不同的布局模式：

1. **登录模式** - 全屏、简洁、专注
2. **应用模式** - 功能完整、信息丰富、操作便捷

这种设计既保证了登录体验的专注性，又保持了应用功能的完整性。
