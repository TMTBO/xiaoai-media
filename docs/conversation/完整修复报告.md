## 核心技术实现

### 对话 API 调用

```python
# URL
url = f"https://userprofile.mina.mi.com/device_profile/v2/conversation?source=dialogu&hardware={hardware}&timestamp={timestamp}&limit=2"

# Cookies（三个都必须）
cookies = {
    "deviceId": device_id,
    "serviceToken": service_token,  # 从 .mi.token 读取
    "userId": str(user_id)           # 从 .mi.token 读取
}

# 发送请求
async with session.get(url, cookies=cookies) as resp:
    data = await resp.json()
    records = json.loads(data["data"]).get("records", [])
```

### 指令识别

```python
# 正则匹配
pattern = r"^(?:播放|打开)(?:歌曲)?(.+)$"

# 过滤控制指令
if not any(x in content for x in ["音量", "暂停", "继续", "停止", "下一首", "上一首"]):
    return {"action": "play", "query": content}
```

### 播放流程

```python
# 1. 搜索音乐
results = await search_music(query)

# 2. 同步播放列表
_playlists[device_id] = {"songs": results, "current": 0}

# 3. 获取 URL
play_info = await _get_play_url_with_fallback(song)

# 4. 播放
await client.play_url(play_info["url"], device_id)
```

## 性能指标

### 响应时间

- 对话检测延迟：0-2 秒（取决于轮询间隔）
- 音乐搜索：~370ms
- URL 获取：~500ms
- 播放调用：~880ms
- **总计：2-4 秒**（从说话到播放）

### 资源消耗

- CPU：可忽略
- 内存：< 1MB
- 网络：每个轮询周期一次请求/设备

---

## 已知限制

1. **轮询延迟**
   - 最快响应 = 轮询间隔
   - 默认 2 秒

2. **小爱回答**
   - 某些对话的回答字段可能为空
   - 这是小米 API 的限制

3. **指令识别**
   - 目前只支持简单播放指令
   - 不支持复杂的歌单、排行榜指令
