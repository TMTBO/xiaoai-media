# 对话监听功能 - 快速开始

## 什么是对话监听？

对话监听功能让你的服务能够"听到"用户对音箱说的话，并自动拦截播放指令，通过你的音乐服务获取 URL 进行播放。

**场景示例：**
- 用户对音箱说："小爱同学，播放周杰伦的晴天"
- 服务自动检测到这个指令
- 自动搜索"周杰伦的晴天"
- 获取音乐 URL
- 在音箱上播放

## 快速启动

### 1. 配置环境变量

编辑 `.env` 文件，确保以下配置存在：

```bash
# 启用对话监听（默认已启用）
ENABLE_CONVERSATION_POLLING=true

# 轮询间隔（秒），默认 2.0
CONVERSATION_POLL_INTERVAL=2.0

# 音乐服务地址（必须配置）
MUSIC_API_BASE_URL=http://localhost:5050
```

### 2. 启动服务

```bash
cd backend
python3 -m uvicorn xiaoai_media.api.main:app --reload
```

### 3. 测试功能

对你的小爱音箱说：

```
小爱同学，播放周杰伦的晴天
```

查看服务日志，你应该能看到：

```
INFO xiaoai_media.conversation — 对话轮询器已启动 (轮询间隔: 2.0秒)
INFO xiaoai_media.conversation — 检测到新对话 (设备 xxx): 播放周杰伦的晴天
INFO xiaoai_media.command_handler — 检测到播放指令: 周杰伦的晴天
INFO xiaoai_media.command_handler — 找到歌曲: 周杰伦 - 晴天 (平台: tx)
INFO xiaoai_media.command_handler — 播放成功: 周杰伦 - 晴天
```

## 支持的指令格式

对话监听器会识别以下格式的播放指令：

- ✅ "播放周杰伦的晴天"
- ✅ "播放晴天"
- ✅ "播放歌曲晴天"
- ✅ "打开周杰伦的歌"
- ❌ "播放音量" (会被过滤)
- ❌ "播放暂停" (会被过滤)
- ❌ "播放下一首" (会被过滤)

## 工作原理

```
┌─────────────┐
│  用户说话   │
│ "播放晴天"  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  小爱音箱   │
│  记录对话   │
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│ ConversationPoller │  ← 每 2 秒轮询一次
│   检测新对话      │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│ CommandHandler  │
│  解析播放指令   │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  搜索音乐服务   │
│  获取 URL       │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  音箱播放音乐   │
└─────────────────┘
```

## 故障排查

### 问题：服务启动但没有检测到对话

**检查：**
1. 确认 `ENABLE_CONVERSATION_POLLING=true`
2. 查看日志是否有 "对话轮询器已启动"
3. 确认小米账号配置正确（MI_USER, MI_PASS_TOKEN）

### 问题：检测到对话但没有播放

**检查：**
1. 确认音乐服务 (MUSIC_API_BASE_URL) 可访问
2. 查看日志中的错误信息
3. 确认搜索到了歌曲

### 问题：播放的不是我想要的歌

**原因：** 搜索结果的第一首歌可能不是你想要的

**解决：** 说话时更明确，例如：
- ❌ "播放晴天" → 可能搜到其他歌手的晴天
- ✅ "播放周杰伦的晴天" → 更精确

## 禁用对话监听

如果不需要这个功能，可以在 `.env` 中设置：

```bash
ENABLE_CONVERSATION_POLLING=false
```

## 性能考虑

- 轮询间隔默认 2 秒，不建议设置太短
- 每次轮询会请求所有设备的对话记录
- 如果有多个设备，会并发请求
