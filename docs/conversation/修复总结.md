# 对话监听功能修复总结

## 问题

**原始问题：** 在后台发送/音箱设备上输入的播放指令没有到本服务中调用 play_music 获取音乐 url

## 解决方案

参考 [xiaomusic](https://github.com/hanxi/xiaomusic) 项目，实现了**对话监听机制**。

## 实现内容

### 1. 新增文件

#### `backend/src/xiaoai_media/conversation.py`
- **ConversationPoller** 类 - 对话轮询器
- 每 2 秒轮询所有音箱的对话记录
- 检测新对话并触发命令处理

#### `backend/src/xiaoai_media/command_handler.py`
- **CommandHandler** 类 - 命令处理器
- 解析播放指令（如"播放周杰伦的晴天"）
- 自动搜索音乐
- 调用 play_music 获取 URL
- 在音箱上播放

### 2. 修改文件

#### `backend/src/xiaoai_media/client.py`
- 添加 `get_latest_ask()` 方法
- 使用小米对话 API 获取对话记录
- API: `https://userprofile.mina.mi.com/device_profile/v2/conversation`

#### `backend/src/xiaoai_media/api/main.py`
- 集成对话轮询器和命令处理器
- 在应用启动时自动启动轮询
- 在应用关闭时停止轮询

#### `backend/src/xiaoai_media/config.py`
- 添加 `ENABLE_CONVERSATION_POLLING` 配置
- 添加 `CONVERSATION_POLL_INTERVAL` 配置

#### `.env.example`
- 添加对话监听相关配置说明

### 3. 测试文件

- `test/test_conversation_monitoring.py` - 对话监听测试
- `test/test_get_conversation.py` - 对话 API 测试

## 工作流程

```
1. 用户对音箱说："小爱同学，播放周杰伦的晴天"
   ↓
2. 小爱云端记录对话
   ↓
3. ConversationPoller 每 2 秒轮询一次
   ↓
4. 检测到新对话："播放周杰伦的晴天"
   ↓
5. CommandHandler 解析指令
   ↓
6. 提取搜索关键词："周杰伦的晴天"
   ↓
7. 调用音乐搜索 API
   ↓
8. 获取搜索结果（20首歌）
   ↓
9. 同步播放列表到服务端
   ↓
10. 调用 play_music 获取第一首歌的 URL
    ↓
11. 在音箱上播放 URL
    ↓
12. ✅ 播放成功！
```

### ✅ 会被拦截并处理

## 关键技术点

### 1. 对话 API 调用

使用小米的对话历史 API：

```python
url = f"https://userprofile.mina.mi.com/device_profile/v2/conversation?source=dialogu&hardware={hardware}&timestamp={timestamp}&limit=2"
cookies = {"deviceId": device_id, "serviceToken": service_token}
```

### 2. 时间戳管理

```python
# 记录每个设备的最后时间戳
self.last_timestamp[device_id] = timestamp

# 只处理新对话
if conversation["time"] > self.last_timestamp[device_id]:
    # 处理新对话
    await handle_command(device_id, query)
```

### 3. 指令解析

```python
# 正则匹配播放指令
pattern = r"^(?:播放|打开)(?:歌曲)?(.+)$"
match = re.match(pattern, query)

# 过滤控制指令
if not any(x in content for x in ["音量", "暂停", "继续", "停止", "下一首", "上一首"]):
    return {"action": "play", "query": content}
```

### 4. 播放流程

```python
# 1. 搜索音乐
results = await search_music(query)

# 2. 同步播放列表
_playlists[device_id] = {"songs": results, "current": 0}

# 3. 获取 URL
play_info = await _get_play_url_with_fallback(song)

# 4. 播放
await client.play_url(play_info["url"], device_id)
```

## 修复的 Bug

### Bug: MiNAService 没有 get_latest_ask 方法

**错误信息：**
```
AttributeError: 'MiNAService' object has no attribute 'get_latest_ask'
```

**原因：** 
- 最初实现错误地假设 `MiNAService` 有 `get_latest_ask` 方法
- 实际上需要直接调用小米的 HTTP API

**修复：**
- 改用直接 HTTP 请求调用对话 API
- 参考 xiaomusic 的实现方式
- 使用 `aiohttp.ClientSession` 发送请求
