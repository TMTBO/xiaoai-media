# 对话监听功能 - 快速参考

## 一句话说明

**自动监听音箱对话，拦截播放指令，通过本服务获取音乐 URL 并播放。**

## 配置（.env）

```bash
ENABLE_CONVERSATION_POLLING=true    # 启用对话监听
CONVERSATION_POLL_INTERVAL=2.0      # 轮询间隔（秒）
MUSIC_API_BASE_URL=http://localhost:5050  # 音乐服务地址
```

## 启动

```bash
cd backend
python3 -m uvicorn xiaoai_media.api.main:app --reload
```

## 测试

对音箱说：
```
小爱同学，播放周杰伦的晴天
```

## 日志示例

```
INFO — 对话轮询器已启动 (轮询间隔: 2.0秒)
INFO — 检测到新对话: 播放周杰伦的晴天
INFO — 检测到播放指令: 周杰伦的晴天
INFO — 找到歌曲: 周杰伦 - 晴天
INFO — 播放成功: 周杰伦 - 晴天
```

## 支持的指令

✅ 播放周杰伦的晴天  
✅ 播放晴天  
✅ 播放歌曲晴天  
✅ 打开周杰伦的歌  

❌ 播放音量 (被过滤)  
❌ 播放暂停 (被过滤)  
❌ 播放下一首 (被过滤)  

## 工作流程

```
用户说话 → 对话记录 → 轮询检测 → 解析指令 
→ 搜索音乐 → 获取URL → 播放
```

## 核心文件

- `conversation.py` - 对话轮询器
- `command_handler.py` - 命令处理器
- `client.py` - 添加 get_latest_ask() 方法
- `main.py` - 集成轮询器

## 故障排查

| 问题 | 检查 |
|------|------|
| 没有检测到对话 | 1. 配置是否启用<br>2. 账号是否正确<br>3. 查看启动日志 |
| 检测到但没播放 | 1. 音乐服务是否运行<br>2. 搜索是否有结果<br>3. 查看错误日志 |
| 播放错误的歌 | 说话时包含歌手名 |
