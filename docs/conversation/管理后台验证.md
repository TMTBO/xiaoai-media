# 管理后台对话记录验证指南

## 快速验证

### 1. 启动服务

```bash
# 终端 1：启动后端
cd backend
python3 -m uvicorn xiaoai_media.api.main:app --reload

# 终端 2：启动前端
cd frontend
npm run dev
```

### 2. 访问管理后台

打开浏览器：`http://localhost:5173`

### 3. 查看对话记录

1. 点击左侧菜单"对话记录"
2. 点击"查询对话"按钮
3. 应该能看到对话列表

### 4. 测试播放指令标记

1. 对音箱说："小爱同学，播放周杰伦的晴天"
2. 等待 3-5 秒
3. 在管理后台点击"查询对话"
4. 应该看到新的对话，并且有绿色标签"播放指令（已拦截）"

### 5. 测试自动刷新

1. 开启"自动刷新"开关
2. 对音箱说话
3. 等待 3 秒，对话记录会自动更新

### 功能特性

✅ **时间戳格式化**
- 今天：显示"今天 HH:MM:SS"
- 昨天：显示"昨天 HH:MM:SS"
- 本周：显示"周X HH:MM:SS"
- 更早：显示完整日期时间

✅ **播放指令识别**
- 自动识别播放指令
- 显示绿色标签"播放指令（已拦截）"
- 表示该指令会被对话监听功能拦截处理

✅ **自动刷新**
- 开关控制
- 每 3 秒自动更新
- 静默刷新（不显示提示消息）

✅ **空状态处理**
- 无对话时显示"暂无对话记录"
- 无回答时显示"暂无回答记录"

✅ **设备选择**
- 支持选择特定设备
- 支持查看所有设备的对话

## API 接口

### 请求

```bash
GET /api/command/conversation?device_id={device_id}
```

参数：
- `device_id`（可选）：设备 ID，不传则使用默认设备

### 响应

```json
{
  "conversations": [
    {
      "timestamp_ms": 1773883896585,
      "question": "播放周杰伦的稻香",
      "content": ""
    }
  ]
}
```

字段说明：
- `timestamp_ms`：时间戳（毫秒）
- `question`：用户的问题
- `content`：小爱的回答（可能为空）

## 故障排查

### 问题：点击查询后没有数据

**检查：**
```bash
# 测试后端接口
curl "http://localhost:8000/api/command/conversation"
```

**可能原因：**
- 后端服务未启动
- 设备未配置
- 小米账号认证失败

### 问题：显示的字段不对

**检查：** 查看浏览器控制台是否有错误

**解决：** 确保前后端字段名称匹配：
- `timestamp_ms`（不是 `time`）
- `question`（不是 `query`）
- `content`（不是 `answer`）

### 问题：自动刷新不工作

**检查：** 查看浏览器控制台是否有错误

**解决：**
- 确保开关已开启
- 检查网络请求是否正常
- 查看是否有 JavaScript 错误
