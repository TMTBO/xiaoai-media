# 播放拦截问题分析与解决方案

## 问题描述

用户说"播放周杰伦的稻香"后，出现以下现象：
1. 小爱先播报"播放失败，请检查网络或者换一个试试"
2. 然后音乐开始播放
3. 播放几秒后停止

## 问题分析

### 1. 小爱播报"播放失败"的原因

**这是正常现象，无法避免。**

当用户说"播放周杰伦的稻香"时，流程如下：

```
用户语音 → 小爱云服务 → 尝试从默认音乐源播放 → 失败 → 播报"播放失败"
                    ↓
                对话记录 API
                    ↓
            我们的轮询器（0.3秒后）
                    ↓
            拦截并搜索音乐（~2.6秒）
                    ↓
                播放音乐
```

**时间线：**
- 0.0 秒：用户说话
- 0.5 秒：小爱云服务尝试播放，失败
- 1.0 秒：小爱播报"播放失败"
- 1.3 秒：我们的轮询器检测到对话记录
- 3.9 秒：我们完成搜索和播放

**结论：** 小爱的"播放失败"是云服务行为，发生在我们拦截之前，无法通过本地拦截阻止。

### 2. 播放几秒后停止的原因

**原因：** 小爱在等待语音回复，如果没有收到回复，会中断当前播放。

**解决方案：** 在播放前发送静默 TTS "." 来占用小爱的响应通道。

### 3. TTS 播报会中断音乐

**问题：** 如果在播放后再发送 TTS（如"拦截成功，正在播放XXX"），会中断音乐播放。

**解决方案：** 只在播放前发送静默 TTS "."，不在播放后发送任何 TTS。

## 最终解决方案

### 修改后的流程

```python
async def _handle_play_command(self, device_id: str, search_query: str):
    # 1. 搜索音乐
    search_results = await self._search_music(search_query)
    
    # 2. 发送静默 TTS 占位（防止小爱中断播放）
    await client.text_to_speech(".", device_id)
    
    # 3. 同步播放列表
    await self._sync_playlist(device_id, songs)
    
    # 4. 播放音乐
    play_result = await self._play_song(device_id, 0)
    
    # 5. 只记录日志，不发送 TTS（避免中断音乐）
    _log.info("播放完成")
```

### 关键点

1. **静默 TTS "."**：占用小爱的响应通道，防止小爱中断播放
2. **不在播放后发送 TTS**：任何 TTS 都会中断音乐播放
3. **只记录日志**：通过日志记录播放状态，不影响用户体验

## 用户体验

修改后的用户体验：

1. 用户说："播放周杰伦的稻香"
2. 小爱说："播放失败，请检查网络或者换一个试试"（云服务行为，无法避免）
3. 音乐开始播放（我们的拦截成功）
4. 音乐持续播放，不会中断

**注意：** 虽然小爱会说"播放失败"，但音乐实际上会正常播放。这是因为我们的拦截机制在云服务失败后接管了播放。

## 技术细节

### player_play_music 数据结构

```python
{
    "startaudioid": "1582971365183456177",
    "music": {
        "payload": {
            "audio_type": "MUSIC",  # 或 "" 表示普通播放
            "audio_items": [{
                "item_id": {
                    "audio_id": "1582971365183456177",
                    "cp": {
                        "album_id": "-1",
                        "episode_index": 0,
                        "id": "355454500",
                        "name": "xiaowei"
                    }
                },
                "stream": {"url": "实际音乐URL"}
            }],
            "list_params": {
                "listId": "-1",
                "loadmore_offset": 0,
                "origin": "xiaowei",
                "type": "MUSIC"
            }
        },
        "play_behavior": "REPLACE_ALL"
    }
}
```

### 适用硬件

使用 `player_play_music` 的硬件型号：
- OH2, OH2P
- LX04, LX05, L05B, L05C, L06, L06A
- X08A, X08C, X08E, X10A, X8F, X4B, X6A

其他硬件使用 `player_play_url`。
