## 重要发现

### 1. 音箱回应"好的"是正常行为
音箱对不同类型的命令有不同的回应方式：
- **动作类命令**：执行具体动作（播放音乐、唱歌等）
- **查询类命令**：回应"好的"表示收到并确认命令
- **静默模式**：不发出任何语音回应

这不是bug，而是小爱音箱的正常工作方式。

### 2. 两种TTS功能的区别
| 功能 | 方法 | aiid | 参数 | 用途 |
|-----|------|------|------|------|
| TTS播报 | text_to_speech() | 1或3 | [text] | 播报通知/消息 |
| 执行命令 | send_command() | 4 | [text, flag] | 执行命令/查询 |

### 3. 响应标志的作用
- `response_flag=1`：音箱会语音回应（"好的"或执行动作）
- `response_flag=0`：音箱静默执行，不语音回应

## 使用建议

### 场景1：播报通知
```python
# 只播报文本，不执行命令
await client.text_to_speech("温度已达到25度", device_id)
await client.text_to_speech("您有新消息", device_id)
```

### 场景2：控制音箱
```python
# 执行动作类命令
await client.send_command("播放音乐", device_id)
await client.send_command("暂停", device_id)
```

### 场景3：查询信息
```python
# 查询类命令（音箱会回应"好的"）
await client.send_command("现在几点", device_id)
await client.send_command("今天天气", device_id)
```

### 场景4：静默控制
```python
# 静默执行，不要语音回应
await client.send_command("关灯", device_id, silent=True)
```
