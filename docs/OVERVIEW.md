# 项目概览

XiaoAI Media 项目的整体介绍和架构说明。

---

## 🎯 项目简介

XiaoAI Media 是一个功能强大的小米/小爱音箱控制系统，旨在提供比官方应用更丰富的功能和更好的用户体验。

### 核心价值

- **功能丰富** - 提供播放列表、定时任务、批量导入等高级功能
- **易于使用** - 现代化的 Web 界面，简单直观
- **高度可定制** - 支持自定义音频源、命令处理等
- **开源免费** - MIT 许可证，完全开源

---

## 🏗️ 架构设计

### 整体架构

```
┌─────────────────────────────────────────┐
│           Web 管理界面 (Vue 3)           │
│  - 设备管理                              │
│  - 播放列表管理                          │
│  - 定时任务管理                          │
│  - 音乐搜索                              │
└──────────────┬──────────────────────────┘
               │ HTTP/SSE
               ↓
┌─────────────────────────────────────────┐
│         REST API (FastAPI)              │
│  - 路由层：HTTP 请求处理                 │
│  - 服务层：业务逻辑                      │
│  - 数据层：数据持久化                    │
└──────────────┬──────────────────────────┘
               │ MiService Protocol
               ↓
┌─────────────────────────────────────────┐
│         小米/小爱音箱设备                 │
│  - 音乐播放                              │
│  - TTS 播报                              │
│  - 语音命令                              │
└─────────────────────────────────────────┘
```

### 分层架构

```
┌─────────────────────────────────────────┐
│      API 路由层 (Routes)                 │
│  - HTTP 请求/响应处理                    │
│  - 参数验证                              │
│  - 错误处理                              │
└──────────────┬──────────────────────────┘
               │ 调用
               ↓
┌─────────────────────────────────────────┐
│       服务层 (Services)                  │
│  - 业务逻辑处理                          │
│  - 数据处理和转换                        │
│  - 外部 API 调用                         │
└──────────────┬──────────────────────────┘
               │ 使用
               ↓
┌─────────────────────────────────────────┐
│    数据层 (Player, Client, Storage)      │
│  - 数据持久化                            │
│  - 设备通信                              │
│  - 文件存储                              │
└─────────────────────────────────────────┘
```

详见：[项目结构](STRUCTURE.md) | [服务层架构](../backend/src/xiaoai_media/services/README.md)

## 🎨 技术栈

### 后端技术

- **Python 3.11+** - 现代 Python 特性
- **FastAPI** - 高性能异步 Web 框架
- **SQLite** - 轻量级数据库
- **aiohttp** - 异步 HTTP 客户端
- **APScheduler** - 定时任务调度
- **MiService** - 小米 IoT 协议库

- **Vue 3** - 渐进式 JavaScript 框架
- **TypeScript** - 类型安全
- **Vite** - 快速构建工具
- **Element Plus** - UI 组件库
- **Vue Router** - 路由管理
- **Axios** - HTTP 客户端

### 部署技术

- **Docker** - 容器化部署
- **Docker Compose** - 多容器编排
- **GitHub Actions** - CI/CD 自动化

## 🔄 数据流

### 播放音乐流程

```
用户操作
  ↓
Web 界面 / API 请求
  ↓
API 路由层（参数验证）
  ↓
服务层（业务逻辑）
  ↓
Player / Client（设备通信）
  ↓
小爱音箱（播放音乐）
```

### 语音命令流程

```
用户语音
  ↓
小爱音箱（识别）
  ↓
对话监听（拦截）
  ↓
语音命令服务（解析）
  ↓
相应服务（执行）
  ↓
小爱音箱（响应）
```

### 定时任务流程

```
创建任务
  ↓
定时任务服务（调度）
  ↓
APScheduler（触发）
  ↓
任务执行器（执行）
  ↓
相应服务（处理）
  ↓
小爱音箱（播放/播报）
```

## 🎯 设计原则

### 1. 关注点分离
- 路由层只处理 HTTP 请求/响应
- 服务层实现业务逻辑
- 数据层处理数据持久化

### 2. 单一职责
- 每个模块只负责一个功能
- 每个类只有一个修改理由
- 每个函数只做一件事

### 3. 代码复用
- 服务层可被多个路由使用
- 组合式函数共享前端逻辑
- 工具函数统一管理

### 4. 易于测试
- 服务层可独立测试
- 模块间依赖清晰
- Mock 容易实现

### 5. 易于扩展
- 新功能易于添加
- 修改影响范围小
- 向后兼容性好

## 🌟 特色功能

### 1. 批量导入
从目录批量导入音频文件，支持：
- 自动识别音频文件
- 自然排序（第1章、第2章、第10章）
- 自动提取元数据
- 一键创建播放列表

### 2. 语音命令智能解析
自动识别和解析语音命令：
- 播放列表关键词匹配
- 排行榜名称识别
- 音乐搜索关键词提取
- 自定义命令处理

### 3. 实时状态监控
- 设备在线状态
- 播放状态（播放中/暂停/停止）
- 当前播放信息
- SSE 实时推送

### 4. 自定义音频源
通过 `get_audio_url()` 函数集成任意音频源：
- 本地 NAS 音乐库
- 自建音乐服务
- 第三方音频平台
- 动态 URL 生成
