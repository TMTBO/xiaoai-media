# 文件导入功能说明

## 功能概述

批量导入功能使用统一的路径选择器，支持灵活的导入方式：
1. **目录导入**：选择一个目录，导入其中的所有音频文件
2. **文件导入**：选择一个或多个音频文件进行导入（支持多选）
3. **混合导入**：可以同时选择目录和文件

所有操作在同一个选择器中完成，无需切换模式。

## 使用方法

### 统一路径选择器

1. 点击"选择路径"输入框或"浏览"按钮打开选择器
2. 在选择器中可以：
   - **选择目录**：点击目录旁的复选框，或点击顶部"选择当前目录"按钮
   - **选择文件**：点击文件旁的复选框（仅音频文件可选）
   - **浏览目录**：点击目录名称或右箭头进入子目录
   - **返回上级**：点击"上级目录"按钮

### 文件类型标识

- **音频文件**：正常显示，可以选择
- **非音频文件**：灰色显示，不可选择
- **目录**：绿色图标，可以选择或进入

### 导入配置

选择完路径后：

1. **如果选择了目录**：
   - 配置"递归扫描子目录"选项
   - 选择要导入的文件格式
   
2. **如果选择了文件**：
   - 直接导入选中的文件
   - 无需配置格式（已经是音频文件）

3. **如果同时选择了目录和文件**：
   - 目录按配置的选项扫描
   - 文件直接导入

4. 点击"开始导入"

## 界面特点

1. **统一体验**：目录和文件在同一个列表中显示
2. **智能标识**：
   - 音频文件：蓝色图标，可选
   - 非音频文件：灰色图标，不可选
   - 目录：绿色图标，可选可进入
3. **可视化反馈**：
   - 选中的项目会高亮显示
   - 显示文件大小
   - 底部显示已选择的数量
4. **标签管理**：
   - 已选项显示为标签
   - 目录标签为绿色
   - 文件标签为默认颜色
   - 可单独移除

## 技术实现

### 后端更改

1. **模型更新** (`playlist_models.py`)
   - `ImportFromDirectoryRequest` 支持 `directory` 和 `files` 参数
   - 两个参数可以同时使用

2. **服务更新** (`playlist_service.py`)
   - `import_from_directory` 方法支持混合模式
   - `browse_directory` 方法返回所有文件（不仅仅是音频文件）
   - 前端负责标识音频文件

3. **API 更新** (`routes/playlist.py`)
   - `/playlists/{playlist_id}/import` 端点支持灵活的请求格式

### 前端更改

1. **PathSelector 组件** (`PathSelector.vue`)
   - 统一的路径选择器
   - 同时显示目录和文件
   - 前端判断音频文件并标识
   - 非音频文件灰色不可选
   - 支持同时选择目录和文件

2. **界面更新** (`PlaylistManager.vue`)
   - 使用统一的 `PathSelector` 组件
   - 根据选择内容动态显示配置选项
   - 简化的逻辑和代码

3. **API 更新** (`api/index.ts`)
   - 更新接口定义支持灵活的请求格式

## 支持的音频格式

- MP3 (.mp3)
- M4A (.m4a)
- FLAC (.flac)
- WAV (.wav)
- OGG (.ogg)
- AAC (.aac)
- WMA (.wma)

## 注意事项

1. 可以同时选择目录和文件
2. 非音频文件在前端标识为灰色不可选
3. 目录模式下，文件扩展名过滤在后端完成
4. 文件模式下，只能选择音频文件
5. 文件路径使用绝对路径
6. 导入的文件会自动按文件名进行自然排序（如果播单类型支持）

## 优势

相比传统的分离选择器：
- **更直观**：所有内容在一个列表中，一目了然
- **更灵活**：可以同时选择目录和文件
- **更简洁**：无需切换模式或界面
- **更智能**：自动标识文件类型，防止误选
- **更高效**：减少了用户操作步骤

## 使用场景

1. **批量导入**：选择整个目录，快速导入所有音频
2. **精确导入**：只选择需要的特定文件
3. **混合导入**：选择一个目录加上其他目录中的几个文件
4. **快速选择**：使用"选择当前目录"按钮快速选择
