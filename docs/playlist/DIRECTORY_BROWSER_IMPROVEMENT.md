# 目录浏览器改进

## 改进内容

### 1. 本地模式：点击路径框打开浏览器

**改进前：**
- 只能通过点击"浏览"按钮打开目录浏览器
- 路径输入框是只读的，点击无反应

**改进后：**
- 点击路径输入框即可打开目录浏览器
- 点击"浏览"按钮也可以打开目录浏览器
- 输入框显示为可点击状态（鼠标指针变化）

### 2. Docker模式：统一使用目录浏览器

**改进前：**
- Docker模式使用下拉选择器，只能选择预定义的目录
- 无法浏览子目录

**改进后：**
- Docker模式与本地模式使用相同的目录浏览器
- 可以浏览 `/data` 目录下的所有子目录
- 提供更灵活的目录选择方式

## 技术实现

### 前端修改

**文件：** `frontend/src/views/PlaylistManager.vue`

1. **统一目录选择界面**
   - 移除了 Docker 模式的下拉选择器
   - 本地和 Docker 模式都使用相同的目录浏览器

2. **增强输入框交互**
   - 添加 `@click` 事件监听器
   - 设置 `cursor: pointer` 样式
   - 保持 `readonly` 属性，防止手动输入错误路径

3. **样式优化**
   ```css
   :deep(.el-input__wrapper) {
       cursor: pointer;
   }
   ```

### 后端支持

后端已经提供了完整的目录浏览功能：

- `browse_directory(path)` - 浏览指定目录
- `is_docker_environment()` - 判断运行环境
- Docker 环境从 `/data` 开始浏览
- 本地环境从用户主目录开始浏览

## 用户体验改进

1. **更直观的操作**
   - 用户可以直接点击输入框来选择目录
   - 减少了操作步骤

2. **统一的交互方式**
   - 本地和 Docker 模式使用相同的界面
   - 降低学习成本

3. **更灵活的目录选择**
   - Docker 模式不再局限于预定义目录
   - 可以浏览任意子目录

## 兼容性

- 保持了原有的 API 接口
- 不影响现有功能
- 向后兼容

## 测试建议

1. **本地模式测试**
   - 点击路径输入框，验证目录浏览器打开
   - 点击"浏览"按钮，验证目录浏览器打开
   - 选择目录后，验证路径正确显示

2. **Docker 模式测试**
   - 验证从 `/data` 目录开始浏览
   - 验证可以进入子目录
   - 验证可以返回父目录
   - 验证选择目录后批量导入功能正常

3. **跨环境测试**
   - 验证本地和 Docker 模式的界面一致性
   - 验证环境提示信息正确显示
