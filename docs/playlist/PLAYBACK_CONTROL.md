# 播单播放控制功能

## 功能概述

播单现在支持完整的播放控制功能，包括：
- 播放、继续、停止控制
- 播放模式切换（列表循环、单曲循环、随机播放）
- 记录当前播放位置
- 语音命令控制

## 播放模式

### 1. 列表循环 (loop)
- 默认模式
- 播放完最后一首后，自动回到第一首继续播放
- 适合连续播放整个播单

### 2. 单曲循环 (single)
- 重复播放当前歌曲
- 适合喜欢的单曲反复欣赏

### 3. 随机播放 (random)
- 随机选择播单中的歌曲播放
- 每次播放下一首时随机选择

## Web界面操作

### 播单列表界面

在播单管理页面，每个播单都有以下控制按钮：

1. **播放** - 从头开始播放播单
2. **继续** - 从上次停止的位置继续播放
3. **停止** - 停止当前播放
4. **播放模式下拉框** - 选择播放模式（列表循环/单曲循环/随机播放）

### 当前位置显示

播单列表会显示当前播放位置，格式为：`当前索引 / 总数`
例如：`3 / 10` 表示当前在第3首，共10首歌曲

## API接口

### 播放播单
```http
POST /api/playlists/{playlist_id}/play
Content-Type: application/json

{
  "device_id": "设备ID（可选）",
  "start_index": 0,
  "announce": true
}
```

### 继续播放
```http
POST /api/playlists/{playlist_id}/continue
Content-Type: application/json

{
  "device_id": "设备ID（可选）",
  "announce": true
}
```

### 停止播放
```http
POST /api/playlists/{playlist_id}/stop?device_id=设备ID
```

### 设置播放模式
```http
POST /api/playlists/{playlist_id}/play-mode
Content-Type: application/json

{
  "play_mode": "loop"  // 可选值: loop, single, random
}
```

### 播放下一首
```http
POST /api/playlists/{playlist_id}/next?device_id=设备ID
```

## 语音命令

### 播放控制命令

- **"继续播放"** - 继续播放当前播单
- **"停止播放"** 或 **"停止"** - 停止播放
- **"下一首"** - 播放下一首（根据当前播放模式）

### 播放模式命令

- **"列表循环"** 或 **"循环播放"** - 切换到列表循环模式
- **"单曲循环"** - 切换到单曲循环模式
- **"随机播放"** 或 **"随机模式"** - 切换到随机播放模式

### 播放播单命令

- **"播放音乐播单"** - 播放名为"音乐"的播单
- **"播放有声书"** - 播放包含"有声书"关键词的播单

## 使用示例

### 场景1：连续播放音乐播单

1. 在Web界面点击"播放"按钮，开始播放
2. 播单会从第一首开始播放
3. 播放完最后一首后自动回到第一首（列表循环模式）

### 场景2：单曲循环

1. 播放到喜欢的歌曲时
2. 在播放模式下拉框选择"单曲循环"
3. 或者语音说："单曲循环"
4. 当前歌曲会重复播放

### 场景3：随机播放

1. 选择播放模式为"随机播放"
2. 或者语音说："随机播放"
3. 每次播放下一首时会随机选择

### 场景4：继续播放

1. 播放到一半时点击"停止"
2. 稍后想继续时，点击"继续"按钮
3. 或者语音说："继续播放"
4. 会从上次停止的位置继续播放

## 技术实现

### 数据模型

播单模型新增字段：
- `play_mode`: 播放模式（loop/single/random）
- `current_index`: 当前播放索引

### 状态管理

使用StateService保存当前播放的播单ID：
- 键格式：`current_playlist_{device_id}`
- 用于语音命令时识别当前播放的播单

### 播放逻辑

1. **播放** - 从指定索引开始播放，更新current_index
2. **继续** - 从current_index位置继续播放
3. **下一首** - 根据play_mode计算下一首索引：
   - loop: `(current_index + 1) % total`
   - single: `current_index`（保持不变）
   - random: `random.randint(0, total-1)`

## 注意事项

1. 播放模式和当前索引会自动保存到播单文件中
2. 语音命令需要先播放一个播单，才能使用"继续播放"、"停止"等命令
3. 切换播放模式会立即生效，影响下一首的播放逻辑
4. 停止播放不会重置current_index，方便继续播放
