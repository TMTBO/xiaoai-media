# 章节命名格式支持

## 概述

批量导入功能支持自动识别目录名中的章节信息，并按章节顺序排序。本文档列出所有支持的章节命名格式。

## 支持的格式

### 1. 中文格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| 第XX章 | `第01章` | 1 |
| 第 XX 章 | `第 1 章` | 1 |
| XX章 | `01章` | 1 |

**示例目录结构**：
```
/data/有声书/
  ├── 第01章/
  ├── 第02章/
  └── 第10章/
```

### 2. 英文格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| Chapter XX | `Chapter 1` | 1 |
| ChapterXX | `Chapter01` | 1 |

**示例目录结构**：
```
/data/audiobook/
  ├── Chapter 1/
  ├── Chapter 2/
  └── Chapter 10/
```

### 3. Stage 格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| stage-XX | `stage-02` | 2 |
| stage_XX | `stage_02` | 2 |
| stage XX | `stage 02` | 2 |
| stageXX | `stage02` | 2 |

**示例目录结构**：
```
/data/game-audio/
  ├── stage-01/
  ├── stage-02/
  └── stage-10/
```

**适用场景**：游戏音频、关卡音效、教程分段等

### 4. Episode 格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| episode-XX | `episode-05` | 5 |
| episode_XX | `episode_05` | 5 |
| episode XX | `episode 05` | 5 |
| episodeXX | `episode05` | 5 |
| ep-XX | `ep-08` | 8 |
| ep_XX | `ep_08` | 8 |
| ep XX | `ep 08` | 8 |
| epXX | `ep08` | 8 |

**示例目录结构**：
```
/data/podcast/
  ├── episode-01/
  ├── episode-02/
  ├── ep-03/
  └── episode-10/
```

**适用场景**：播客、电视剧音频、系列节目等

### 5. 纯数字格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| XX | `01` | 1 |
| X | `1` | 1 |

**示例目录结构**：
```
/data/music/
  ├── 01/
  ├── 02/
  └── 10/
```

### 6. 前缀数字格式

| 格式 | 示例 | 提取的章节号 |
|------|------|-------------|
| XX-标题 | `01-序章` | 1 |
| XX_标题 | `01_序章` | 1 |
| XX 标题 | `01 序章` | 1 |

**示例目录结构**：
```
/data/audiobook/
  ├── 01-序章/
  ├── 02-第一章/
  └── 10-尾声/
```

## 排序规则

### 基本规则

1. 有章节号的目录按章节号升序排列
2. 同一章节号的目录按目录名字母顺序排列
3. 无章节号的目录排在最后
4. 同一目录内的文件按文件名自然排序

### 排序示例

**输入目录**：
```
/data/
  ├── 第10章/
  ├── stage-02/
  ├── 音乐/
  ├── 第01章/
  ├── episode-05/
  └── Chapter 3/
```

**排序结果**：
```
1. 第01章/        (章节1)
2. stage-02/      (章节2)
3. Chapter 3/     (章节3)
4. episode-05/    (章节5)
5. 第10章/        (章节10)
6. 音乐/          (无章节号，排最后)
```

## 混合格式支持

系统支持在同一次导入中混合使用不同的章节格式，所有格式都会被统一按章节号排序。

**示例**：
```
/data/mixed/
  ├── Chapter 1/
  ├── stage-02/
  ├── 第03章/
  ├── episode-04/
  └── 05/

排序结果：
1. Chapter 1/     (章节1)
2. stage-02/      (章节2)
3. 第03章/        (章节3)
4. episode-04/    (章节4)
5. 05/            (章节5)
```

## 不支持的格式

以下格式**不会**被识别为章节信息：

- 包含非数字的复杂格式：`Part-A`、`Section-One`
- 中间位置的数字：`音频01集`、`track-01-final`
- 罗马数字：`Chapter I`、`第一章`

对于这些格式，目录会被视为无章节号，排在最后，并按目录名自然排序。

## 自定义需求

如果你的目录命名格式不在支持列表中，可以：

1. 重命名目录以符合支持的格式
2. 提交 Issue 请求添加新格式支持
3. 修改 `playlist_service.py` 中的 `_extract_directory_sort_key` 方法添加自定义正则表达式

## 技术实现

章节号提取使用正则表达式匹配，按以下顺序尝试：

```python
chapter_patterns = [
    r'第\s*(\d+)\s*章',      # 第01章、第 1 章
    r'chapter\s*(\d+)',      # Chapter 1、Chapter01
    r'stage[\-_\s]*(\d+)',   # stage-02、stage_02、stage 02、stage02
    r'episode[\-_\s]*(\d+)', # episode-02、episode_02、episode 02
    r'ep[\-_\s]*(\d+)',      # ep-02、ep_02、ep 02
    r'^(\d+)\s*章',          # 01章、1 章
    r'^(\d+)[_\-\s]',        # 01-、01_、01 开头
    r'^(\d+)$',              # 纯数字目录名
]
```

匹配时不区分大小写（使用 `.lower()`）。
