# 目录选择器功能说明

## 功能概述

为批量导入功能添加了三种导入方式，让用户可以更灵活地选择如何导入音频文件。

## 三种导入模式

### 1. 从服务器路径导入（推荐用于Docker和服务器环境）

#### Docker模式
- **特点**：从下拉列表选择已挂载的目录
- **优势**：简单直观，无需输入路径
- **适用场景**：Docker部署环境

**使用步骤：**
1. 在 docker-compose.yml 中配置 volume 挂载
2. 重启容器
3. 从下拉列表选择目录
4. 配置导入选项
5. 点击"开始导入"

#### 本地模式
- **特点**：手动输入或选择目录路径
- **优势**：灵活，支持任意路径
- **适用场景**：本地开发环境

**使用步骤：**
1. 点击"选择目录"按钮（会显示提示信息）
2. 或直接输入完整路径（如：`/Users/username/Music`）
3. 配置导入选项
4. 点击"开始导入"

**注意事项：**
- 由于浏览器安全限制，"选择目录"按钮无法获取完整路径
- 建议直接手动输入完整路径
- 或使用"从浏览器上传"模式

### 2. 从浏览器上传（推荐用于少量文件）

- **特点**：直接从浏览器上传文件
- **优势**：无需配置服务器路径，即传即用
- **适用场景**：少量文件（< 100个），临时导入

**使用步骤：**
1. 切换到"从浏览器上传"模式
2. 拖拽文件到上传区域，或点击选择文件
3. 支持多选文件
4. 点击"上传并导入"

**特点：**
- 支持拖拽上传
- 支持多文件选择
- 自动提取文件信息（标题、艺术家、专辑）
- 实时显示已选文件数量

**限制：**
- 适合少量文件（建议 < 100个）
- 大量文件建议使用服务器路径导入
- 文件会临时存储在浏览器内存中

## 使用场景对比

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| Docker部署 | 服务器路径（Docker模式） | 简单直观，从下拉列表选择 |
| 本地开发 | 服务器路径（本地模式） | 支持大量文件，性能好 |
| 临时导入少量文件 | 浏览器上传 | 无需配置，即传即用 |
| 大量文件（>100） | 服务器路径 | 性能好，不占用浏览器内存 |
| 测试功能 | 浏览器上传 | 快速测试，无需配置 |

## 技术实现

### 服务器路径导入

**Docker模式：**
```typescript
// 调用API获取可用目录
const data = await api.getAvailableDirectories()
availableDirectories.value = data.directories

// 用户从下拉列表选择
importForm.value.directory = selectedPath

// 调用后端API导入
await api.importFromDirectory(playlistId, {
  directory: importForm.value.directory,
  recursive: true,
  file_extensions: ['.mp3', '.flac']
})
```

**本地模式：**
```typescript
// 用户手动输入路径
importForm.value.directory = '/Users/username/Music'

// 或点击选择目录按钮（会显示提示）
directoryInput.value.click()

// 调用后端API导入
await api.importFromDirectory(playlistId, {
  directory: importForm.value.directory,
  recursive: true,
  file_extensions: ['.mp3', '.flac']
})
```

### 浏览器上传

```typescript
// 用户选择文件
<el-upload
  :auto-upload="false"
  :on-change="handleFileChange"
  multiple
  accept=".mp3,.m4a,.flac,.wav,.ogg,.aac,.wma"
  drag
/>

// 处理文件
function handleFileChange(file: any, fileList: any[]) {
  uploadFileList.value = fileList
}

// 上传并导入
async function handleUploadImport() {
  const items: PlaylistItem[] = []
  
  for (const fileItem of uploadFileList.value) {
    const file = fileItem.raw as File
    
    // 提取文件信息
    const fileName = file.name
    const nameWithoutExt = fileName.substring(0, fileName.lastIndexOf('.'))
    
    // 创建临时URL
    const fileUrl = URL.createObjectURL(file)
    
    items.push({
      title: nameWithoutExt,
      url: fileUrl,
      // ... 其他信息
    })
  }
  
  // 添加到播单
  await api.addPlaylistItems(playlistId, { items })
}
```

## 浏览器安全限制说明

### 为什么无法获取完整路径？

出于安全考虑，现代浏览器不允许JavaScript获取用户文件系统的完整路径。这是为了防止恶意网站访问用户的文件系统。

### 解决方案

1. **Docker模式（推荐）**
   - 在服务器端配置目录挂载
   - 前端从预定义的目录列表中选择
   - 无需获取客户端路径

2. **手动输入路径**
   - 用户知道完整路径
   - 直接输入到文本框
   - 适合本地开发环境

3. **浏览器上传模式**
   - 不需要路径
   - 直接上传文件内容
   - 适合少量文件

## 文件信息提取

### 服务器路径导入

从文件路径自动提取：

```
/data/music/周杰伦/叶惠美/以父之名.mp3
    ↓
标题: 以父之名
艺术家: 周杰伦
专辑: 叶惠美
```

### 浏览器上传

从文件名和相对路径提取：

```
文件: 周杰伦/叶惠美/以父之名.mp3
    ↓
标题: 以父之名
艺术家: 周杰伦
专辑: 叶惠美
```

如果只有文件名：

```
文件: 以父之名.mp3
    ↓
标题: 以父之名
艺术家: (空)
专辑: (空)
```

## 性能考虑

### 服务器路径导入

- **优势**：
  - 不占用浏览器内存
  - 支持大量文件
  - 扫描速度快
  - 适合批量导入

- **性能**：
  - 100个文件：< 2秒
  - 1000个文件：< 10秒
  - 10000个文件：< 60秒

### 浏览器上传

- **优势**：
  - 无需配置服务器
  - 即传即用
  - 适合临时导入

- **限制**：
  - 占用浏览器内存
  - 大量文件可能导致浏览器卡顿
  - 建议 < 100个文件

- **性能**：
  - 10个文件：< 1秒
  - 50个文件：< 5秒
  - 100个文件：< 10秒

## 常见问题

### Q1: 点击"选择目录"按钮后为什么显示提示？

A: 由于浏览器安全限制，无法获取完整的文件系统路径。建议：
- 使用Docker模式（从下拉列表选择）
- 手动输入完整路径
- 使用"从浏览器上传"模式

### Q2: 上传模式支持多少个文件？

A: 理论上没有限制，但建议：
- 少量文件（< 50）：体验最佳
- 中等数量（50-100）：可以接受
- 大量文件（> 100）：建议使用服务器路径导入

### Q3: 上传的文件存储在哪里？

A: 上传模式下，文件会：
1. 临时存储在浏览器内存中（使用 Blob URL）
2. 文件信息保存到播单中
3. 播放时从 Blob URL 读取

注意：刷新页面后 Blob URL 会失效，建议使用服务器路径导入以获得持久化存储。

### Q4: 如何选择合适的导入模式？

A: 根据场景选择：
- **Docker部署** → 服务器路径（Docker模式）
- **本地开发，大量文件** → 服务器路径（本地模式）
- **临时测试，少量文件** → 浏览器上传
- **不想配置服务器** → 浏览器上传

## 最佳实践

### Docker环境

1. 在 docker-compose.yml 中配置 volume：
   ```yaml
   volumes:
     - ./data:/data
     - /path/to/music:/data/music
   ```

2. 重启容器：
   ```bash
   docker-compose restart
   ```

3. 使用前端：
   - 选择"从服务器路径导入"
   - 从下拉列表选择 "music"
   - 勾选"递归扫描"
   - 点击"开始导入"

### 本地环境

1. 确定音乐目录的完整路径
2. 使用前端：
   - 选择"从服务器路径导入"
   - 手动输入路径（如：`/Users/username/Music`）
   - 勾选"递归扫描"
   - 点击"开始导入"

### 临时导入

1. 使用前端：
   - 选择"从浏览器上传"
   - 拖拽文件到上传区域
   - 点击"上传并导入"
