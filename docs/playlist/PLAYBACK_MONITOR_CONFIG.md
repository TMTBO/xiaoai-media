# 播放监控配置说明

## 配置项

播放监控功能支持以下配置项，可以通过 `user_config.py` 或管理后台进行配置：

### ENABLE_PLAYBACK_MONITOR

- 类型：`bool`
- 默认值：`True`
- 说明：是否启用播放监控（自动播放下一曲）
- 作用：设置为 `True` 时，系统会监控设备播放状态，在一曲播放完成后自动播放下一曲

### PLAYBACK_MONITOR_INTERVAL

- 类型：`float`
- 默认值：`3.0`
- 取值范围：`0.5` - `60.0` 秒
- 说明：播放监控轮询间隔（秒）
- 建议：
  - 2-5 秒：适合大多数场景，响应及时且不会过度占用资源
  - 过短（< 2秒）：会增加设备负担，可能导致请求过于频繁
  - 过长（> 5秒）：会导致切歌延迟明显

## 配置方式

### 方式一：通过 user_config.py 配置

编辑 `user_config.py` 文件：

```python
# ============================================
# 播放监控配置
# ============================================

# 启用播放监控（自动播放下一曲）
ENABLE_PLAYBACK_MONITOR = True

# 播放监控轮询间隔（秒）
PLAYBACK_MONITOR_INTERVAL = 3.0
```

### 方式二：通过管理后台配置

1. 访问管理后台的配置页面
2. 找到"播放监控配置"部分
3. 修改以下选项：
   - "启用播放监控"：勾选或取消勾选
   - "播放监控轮询间隔"：输入数值（0.5-60 秒）
4. 点击"保存"按钮

配置会立即生效，无需重启服务。

## 使用场景

### 启用播放监控

适用于以下场景：
- 播放播单时希望自动连续播放
- 需要列表循环、单曲循环或随机播放功能
- 希望获得类似音乐播放器的连续播放体验

### 禁用播放监控

适用于以下场景：
- 只需要播放单曲，不需要自动切歌
- 设备资源有限，希望减少后台轮询
- 使用其他方式控制播放流程

## 注意事项

1. 播放监控只对通过播单播放的音乐生效
2. 直接通过语音指令播放的单曲不会触发自动播放下一曲
3. 播放监控会在应用启动时自动启动，在应用关闭时自动停止
4. 修改配置后，新的设置会在下次轮询时生效
