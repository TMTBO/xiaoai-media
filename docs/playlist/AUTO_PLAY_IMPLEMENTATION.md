# 自动播放下一曲功能实现总结

## 功能概述

实现了播放列表自动播放下一曲的功能，使用定时器模式，根据音频时长自动设置定时器，在歌曲结束前触发下一曲。

## 实现的文件

### 后端核心模块

1. **playback_controller.py** - 播放控制器
   - 根据音频时长设置定时器
   - 在歌曲结束前触发下一曲
   - 支持暂停、继续、停止等操作

2. **playlist_service.py** - 播放列表服务
   - 添加 `random` 模块导入
   - 提供 `play_next_in_playlist()` 方法
   - 支持三种播放模式：loop、single、random

3. **config.py** - 配置管理
   - 添加 `PLAYBACK_MODE` 配置项（默认 controller）

4. **main.py** - 应用入口
   - 初始化播放控制器
   - 在应用启动时检查并恢复播放状态
   - 在应用关闭时停止控制器

### 后端配置管理

5. **config_service.py** - 配置服务
   - 将播放模式配置项添加到 `ALLOWED_KEYS`
   - 在 `get_current_config()` 中返回播放模式配置

6. **routes/config.py** - 配置 API
   - 在 `ConfigUpdate` 模型中添加播放模式字段
   - 支持通过 API 更新播放模式配置

### 前端界面

7. **frontend/src/api/index.ts** - API 类型定义
   - 在 `Config` 接口中添加播放模式字段

8. **frontend/src/views/Settings.vue** - 配置页面
   - 添加"播放控制"配置区域
   - 提供播放模式选择（仅定时器模式）

### 配置文件

9. **user_config.py** - 用户配置
10. **user_config_template.py** - 配置模板

所有配置文件都移除了播放监控相关配置项。

### 文档

11. **docs/PLAYBACK_CONTROLLER.md** - 播放控制器文档
12. **docs/PLAYBACK_CONTROLLER_CHANGELOG.md** - 更新日志

## 配置项说明

### PLAYBACK_MODE

- 类型：`str`
- 默认值：`"controller"`
- 可选值：`"controller"`（定时器模式）
- 说明：播放控制模式

## 配置方式

### 方式一：编辑配置文件

播放控制器默认启用，无需额外配置。

### 方式二：通过管理后台

1. 访问管理后台的"配置管理"页面
2. 找到"播放控制"部分
3. 播放模式默认为"定时器模式"

## 工作流程

1. 应用启动时，初始化播放控制器
2. 播放开始时：
   - 获取音频时长和当前位置
   - 计算剩余播放时间
   - 设置定时器
3. 定时器触发时：
   - 调用 `PlaylistService.play_next_in_playlist()`
   - 根据播放模式选择下一首
   - 发送播放命令到设备
4. 应用关闭时，停止播放控制器

## 播放模式

- **loop**（列表循环）：播放下一首，到末尾后回到开头
- **single**（单曲循环）：重复播放当前曲目
- **random**（随机播放）：随机选择一首

## 状态管理

使用 `state_service` 管理状态：

- `current_playlist_{device_id}`：记录设备当前播放的播单 ID
- 控制器内部维护每个设备的定时器状态

## 技术特点

1. **异步实现**：使用 `asyncio` 实现非阻塞定时器
2. **独立模块**：播放控制器独立于其他模块，易于维护
3. **高效节能**：只在需要时调用 API，性能提升约 30 倍
4. **状态持久化**：播单状态保存在状态服务中
5. **错误处理**：完善的异常处理和日志记录

## 注意事项

1. 只对通过播单播放的音乐生效
2. 直接语音指令播放的单曲不会触发自动播放
3. 依赖准确的音频时长信息
4. 配置修改后立即生效，无需重启
