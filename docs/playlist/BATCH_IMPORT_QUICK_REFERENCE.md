# 批量导入快速参考

## 基本使用

### 1. 打开批量导入

在播单管理页面：
1. 选择一个播单
2. 点击"项目"按钮
3. 点击"批量导入"按钮

### 2. 选择目录

**方式一：点击输入框**
- 直接点击路径输入框
- 自动打开目录浏览器

**方式二：点击浏览按钮**
- 点击"浏览"按钮
- 打开目录浏览器

**目录浏览器操作：**
- 点击文件夹进入子目录
- 点击"上级目录"返回父目录
- 点击"选择当前目录"确认选择

### 3. 配置导入选项

**扫描选项：**
- ☑️ 递归扫描子目录：扫描所有子文件夹
- ☐ 递归扫描子目录：只扫描当前文件夹

**文件格式：**
- ☑️ MP3
- ☑️ M4A
- ☑️ FLAC
- ☑️ WAV
- ☑️ OGG
- ☑️ AAC
- ☐ WMA

### 4. 开始导入

点击"开始导入"按钮，等待导入完成。

### 5. 查看结果

导入完成后会显示：
- ✅ 成功导入数量
- ⏭️ 跳过数量
- 📁 扫描总数
- 🎵 播单总数

**2 秒后对话框自动关闭**

## 文件排序规则

### 音乐播单（不排序）

**播单类型：** 音乐 (music)

**行为：** 保持文件系统原始顺序

**示例：**
```
导入前：
- 03-歌曲C.mp3
- 01-歌曲A.mp3
- 02-歌曲B.mp3

导入后：
1. 03-歌曲C
2. 01-歌曲A
3. 02-歌曲B
```

### 有声书播单（自动排序）

**播单类型：** 有声书 (audiobook)

**行为：** 按章节编号排序

**支持格式：**
- `第01章.mp3`, `第1章.mp3`
- `001.mp3`, `01.mp3`, `1.mp3`
- `Chapter 1.mp3`, `Chapter 01.mp3`
- `001-标题.mp3`, `1.标题.mp3`

**示例：**
```
导入前：
- 第10章.mp3
- 第2章.mp3
- 第1章.mp3

导入后：
1. 第1章
2. 第2章
3. 第10章
```

### 播客播单（自动排序）

**播单类型：** 播客 (podcast)

**行为：** 按集数排序

**支持格式：**
- `Episode 1.mp3`, `Episode 01.mp3`
- `EP01.mp3`, `EP1.mp3`
- `001.mp3`, `01.mp3`

**示例：**
```
导入前：
- Episode 5.mp3
- Episode 10.mp3
- Episode 2.mp3

导入后：
1. Episode 2
2. Episode 5
3. Episode 10
```

### 广播剧播单（自动排序）

**播单类型：** 广播剧 (radio_drama)

**行为：** 按集数排序

**规则同播客**

## 编号格式识别

### 中文格式

| 格式 | 示例 | 识别结果 |
|------|------|---------|
| 第X章 | 第01章.mp3 | 1 |
| 第X集 | 第2集.mp3 | 2 |
| 第X节 | 第3节.mp3 | 3 |
| X章 | 1章.mp3 | 1 |

### 英文格式

| 格式 | 示例 | 识别结果 |
|------|------|---------|
| Chapter X | Chapter 1.mp3 | 1 |
| Episode X | Episode 01.mp3 | 1 |
| EP X | EP 5.mp3 | 5 |

### 数字格式

| 格式 | 示例 | 识别结果 |
|------|------|---------|
| 纯数字 | 001.mp3 | 1 |
| 数字- | 01-标题.mp3 | 1 |
| 数字. | 1.标题.mp3 | 1 |
| 数字_ | 1_标题.mp3 | 1 |
| 数字空格 | 1 标题.mp3 | 1 |

### 特殊情况

| 情况 | 处理方式 |
|------|---------|
| 无编号 | 排在最后 |
| 多个编号 | 使用第一个 |
| 编号不连续 | 按实际编号排序 |

## 环境差异

### 本地模式

**特点：**
- 从用户主目录开始浏览
- 可以访问整个文件系统
- 可以直接输入路径

**默认起始目录：**
- 用户主目录
- Music 文件夹
- Documents/Music 文件夹
- Downloads 文件夹

### Docker 模式

**特点：**
- 从 /data 目录开始浏览
- 只能访问挂载的 volume
- 使用相同的目录浏览器

**默认起始目录：**
- /data

## 常见问题

### Q: 为什么有些文件被跳过？

**可能原因：**
1. 文件格式不在选择的列表中
2. 文件无法读取（权限问题）
3. 文件已损坏

**解决方法：**
- 检查文件扩展名
- 检查文件权限
- 查看跳过文件列表

### Q: 如何让音乐也按编号排序？

**方法：**
1. 创建播单时选择"其他"类型
2. 或者手动调整播单项顺序

### Q: 排序不正确怎么办？

**检查：**
1. 文件名格式是否符合支持的格式
2. 播单类型是否正确
3. 编号是否有重复

**解决：**
- 重命名文件使用标准格式
- 修改播单类型
- 手动调整顺序

### Q: Docker 模式看不到我的文件？

**原因：**
- 目录没有挂载到 /data

**解决：**
```yaml
# docker-compose.yml
volumes:
  - /path/to/your/music:/data/music
```

### Q: 导入后对话框没有自动关闭？

**可能原因：**
1. 导入失败
2. 没有文件被导入

**检查：**
- 查看导入结果
- 检查错误信息

## 最佳实践

### 1. 文件命名规范

**有声书：**
```
第01章 开篇.mp3
第02章 发展.mp3
第03章 高潮.mp3
```

**播客：**
```
Episode 001 - 标题.mp3
Episode 002 - 标题.mp3
Episode 003 - 标题.mp3
```

**音乐：**
```
艺术家 - 歌曲名.mp3
```

### 2. 目录结构

**有声书：**
```
有声书名/
  第01章.mp3
  第02章.mp3
  第03章.mp3
```

**音乐：**
```
艺术家/
  专辑/
    01-歌曲1.mp3
    02-歌曲2.mp3
```

### 3. 导入策略

**大量文件：**
- 开启递归扫描
- 按专辑/系列分别导入

**少量文件：**
- 关闭递归扫描
- 直接选择目标文件夹

## 性能提示

### 大量文件导入

**建议：**
- 分批导入（每次 < 1000 个文件）
- 关闭不需要的文件格式
- 使用递归扫描时注意目录深度

**预期时间：**
- 100 个文件：< 5 秒
- 1000 个文件：< 30 秒
- 10000 个文件：< 5 分钟

### 网络文件系统

**注意：**
- NFS/SMB 挂载可能较慢
- 建议先复制到本地
- 或使用 Docker volume
