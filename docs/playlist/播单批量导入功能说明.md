# 播单批量导入功能说明

## 功能简介

现在可以从指定目录批量导入音频文件到播单中了！支持本地模式和Docker模式，自动提取文件信息。

## 主要功能

### 1. 自动识别运行环境
- 本地开发：可以使用任意本地路径
- Docker部署：从挂载的目录中选择

### 2. 灵活的导入选项
- 可以只扫描当前目录
- 可以递归扫描所有子目录
- 可以指定要导入的文件格式

### 3. 自动提取信息
- 文件名 → 歌曲标题
- 父目录 → 专辑名
- 祖父目录 → 艺术家名

## 快速使用

### Docker环境（推荐）

#### 第一步：配置目录挂载

编辑 `docker-compose.yml`，添加你的音乐目录：

```yaml
volumes:
  - ./data:/data
  - /你的音乐目录路径:/data/music  # 添加这一行
```

例如：
```yaml
volumes:
  - ./data:/data
  - /mnt/nas/music:/data/music
  - /mnt/nas/audiobooks:/data/audiobooks
```

#### 第二步：重启容器

```bash
docker-compose restart
```

#### 第三步：使用API导入

```bash
# 1. 查看可用目录
curl http://localhost:8000/api/playlists/directories

# 2. 创建播单
curl -X POST http://localhost:8000/api/playlists \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的音乐",
    "type": "music",
    "description": "从本地导入"
  }'

# 3. 批量导入（记得替换 {playlist_id}）
curl -X POST http://localhost:8000/api/playlists/{playlist_id}/import \
  -H "Content-Type: application/json" \
  -d '{
    "directory": "/data/music",
    "recursive": true
  }'
```

### 本地环境

直接使用本地路径：

```bash
curl -X POST http://localhost:8000/api/playlists/{playlist_id}/import \
  -H "Content-Type: application/json" \
  -d '{
    "directory": "/Users/你的用户名/Music",
    "recursive": true
  }'
```

## 支持的音频格式

默认支持以下格式：
- MP3 (`.mp3`)
- M4A (`.m4a`)
- FLAC (`.flac`)
- WAV (`.wav`)
- OGG (`.ogg`)
- AAC (`.aac`)
- WMA (`.wma`)

可以通过 `file_extensions` 参数自定义。

## 目录结构示例

假设你的音乐目录结构是这样的：

```
/data/music/
├── 周杰伦/
│   ├── 叶惠美/
│   │   ├── 以父之名.mp3
│   │   └── 晴天.mp3
```

## API说明

### 查看可用目录

```
GET /api/playlists/directories
```

返回：
```json
{
  "directories": [
    {
      "path": "/data/music",
      "name": "music",
      "is_docker": true
    }
  ],
  "is_docker": true,
  "message": "Docker模式：从列表中选择目录"
}
```

### 批量导入

```
POST /api/playlists/{playlist_id}/import
```

请求参数：
```json
{
  "directory": "/data/music",           // 必填：目录路径
  "recursive": true,                    // 可选：是否递归扫描，默认false
  "file_extensions": [".mp3", ".flac"]  // 可选：文件格式，默认全部
}
```

返回：
```json
{
  "imported": 50,              // 成功导入的文件数
  "skipped": 2,                // 跳过的文件数
  "total_scanned": 100,        // 扫描的总文件数
  "playlist_total_items": 50   // 播单中的总项目数
}
```

## 常见问题

### 1. Docker模式下找不到目录？

确保：
- 已在 docker-compose.yml 中配置了volume
- 已重启容器
- 使用 `/api/playlists/directories` 查看实际可用的目录

### 2. 没有文件被导入？

可能原因：
- 文件格式不匹配（检查 file_extensions 参数）
- 目录为空
- 没有开启递归扫描但文件在子目录中

### 3. 权限错误？

- 检查目录权限
- Docker环境：确保挂载的目录有读取权限

## 完整示例

```bash
#!/bin/bash

# 创建播单
PLAYLIST_ID=$(curl -s -X POST http://localhost:8000/api/playlists \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的音乐收藏",
    "type": "music",
    "voice_keywords": ["音乐", "歌曲"]
  }' | jq -r '.id')

echo "播单ID: $PLAYLIST_ID"

# 导入文件
curl -X POST "http://localhost:8000/api/playlists/$PLAYLIST_ID/import" \
  -H "Content-Type: application/json" \
  -d '{
    "directory": "/data/music",
    "recursive": true
  }' | jq

# 查看结果
curl -s "http://localhost:8000/api/playlists/$PLAYLIST_ID" | jq

# 播放
curl -X POST "http://localhost:8000/api/playlists/$PLAYLIST_ID/play" \
  -H "Content-Type: application/json" \
  -d '{
    "start_index": 0,
    "announce": true
  }' | jq
```

## 注意事项

1. **Docker环境必须先挂载目录**
   - 修改 docker-compose.yml
   - 重启容器才能生效

2. **大量文件导入可能较慢**
   - 建议分批导入
   - 或者不使用递归模式

3. **文件路径使用 file:// 协议**
   - 播放设备需要能访问该路径
   - Docker环境中路径在容器内部
